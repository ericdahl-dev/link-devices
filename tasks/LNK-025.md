## Goal

Let the existing **screenless** X32Link firmware be flashed and run on the
**Waveshare ESP32-S3-Touch-LCD-1.47** with a working per-beat LED — **without**
any display/touch/LVGL code. This is the small, standalone board-support slice
that is *not* LNK-014: LNK-014/015/016 are the full on-device display+touch UI;
this ticket is just "make the tempo→OSC bridge boot and blink on this hardware
in hand today."

The wrinkle this solves: the board has **no addressable WS2812** (that's the
non-touch ESP32-S3-LCD-1.47 dongle — different PCB; see LNK-014 Progress). Its
only LEDs are a charger-status LED and an always-on power LED, neither
MCU-driven, and GPIO38 (where the dongle's WS2812 lives) is **LCD_SCL** here.
So the beat indicator is repurposed onto the **LCD backlight enable (GPIO46,
LCD_BL, active-HIGH via NPN)**, driven as a plain digital LED — the panel is
never initialized, so it sits dark and the whole screen strobes on each beat.

## Notes

- Verified hardware facts (unit STA MAC `a4:cb:8f:da:df:d0`): ESP32-S3R8,
  16MB flash, 8MB octal (OPI) PSRAM, native USB-CDC. Full schematic-confirmed
  pinout now lives in **LNK-014** (patched same session) — don't duplicate it,
  reference it.
- Beat-LED path is the existing `led_task` in `X32Link.ino` (LNK-021 phase-lock
  + LNK-023 WiFi-down triple-blink); this ticket only points `LED_PIN_NUM` at a
  new pin, no LED-logic changes.
- Serial-over-native-USB is unreliable on this board (documented in
  `esp32/AGENTS.md`); use the web `/status` endpoint to confirm liveness/BPM.
  Firmware advertises no mDNS — find the unit by its Espressif MAC in the
  router's DHCP table (or ARP-scan from a host on the same WiFi).
