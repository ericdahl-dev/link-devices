## Goal

Add the Ableton Link SDK as a git submodule, write the `link_bridge.h`/`link_bridge.cpp`
C wrapper, and prove that the sketch compiles for XIAO_ESP32S3. This is a spike —
the primary output is confirmed compiler flags and a documented list of any
patches needed to the SDK headers.

## Acceptance Criteria

- [x] `git submodule add https://github.com/Ableton/link esp32/lib/link` succeeds
- [x] `esp32/X32Link/link_bridge.h` declares the C interface (see Notes)
- [x] `esp32/X32Link/link_bridge.cpp` compiles using `abl_link` extension
- [x] `build_flags` or `compiler.cpp.extra_flags` documented in Notes → Progress
- [x] Sketch (X32Link.ino + link_bridge.cpp) compiles for XIAO_ESP32S3 with no errors
- [x] Binary size recorded (flash % and DRAM %) — must leave headroom for OSC + WiFi buffers

## Notes

**Firmware scope**: `esp32/X32Link/` only. Do NOT touch `esp32/X32_emulator/`.

**SDK layout** (after submodule add):
```
esp32/lib/link/
  include/ableton/Link.hpp          main C++ API
  extensions/abl_link/include/
    abl_link.h                      C interface (what we expose to .ino)
    abl_link.hpp                    C++ impl backing the C interface
```

**link_bridge.h** — C interface the sketch uses:
```c
#pragma once
#ifdef __cplusplus
extern "C" {
#endif

void  link_init(double bpm);    // create Link instance, set start BPM
void  link_enable(int enabled); // 1 = join network session, 0 = leave
double link_get_bpm(void);      // capture current session tempo
int   link_peer_count(void);    // number of peers (including self)

#ifdef __cplusplus
}
#endif
```

**link_bridge.cpp** — wraps `abl_link`:
```cpp
#include "link_bridge.h"
#include <abl_link.h>   // from extensions/abl_link/include/

static abl_link_ref s_link = nullptr;

void link_init(double bpm) {
    s_link = abl_link_create(bpm);
}
void link_enable(int enabled) {
    abl_link_enable(s_link, (bool)enabled);
}
double link_get_bpm(void) {
    abl_link_session_state_ref state = abl_link_capture_app_session_state(s_link);
    double bpm = abl_link_tempo(state);
    abl_link_destroy_session_state(state);
    return bpm;
}
int link_peer_count(void) {
    return (int)abl_link_num_peers(s_link);
}
```

**Include path** — arduino-cli needs the SDK headers on the include path.
Likely via a `platform.local.txt` or `build_flags` entry pointing to:
- `esp32/lib/link/include`
- `esp32/lib/link/extensions/abl_link/include`
- `esp32/lib/link/modules/asio-standalone/asio/include`

The exact mechanism depends on the Arduino ESP32 core version (3.3.10). Document
what works in the Progress section.

**Expected compiler flags** (may need adjustment):
```
-DLINK_PLATFORM_POSIX=1
-DASIO_STANDALONE
-DASIO_NO_DEPRECATED
-std=gnu++17
```

**Memory budget**: PSRAM is 8 MB but Link's state machine runs in DRAM.
The abl_link instance is ~8–16 KB. If DRAM is tight, investigate
`EXT_RAM_BSS_ATTR` for static Link buffers (same technique as ESP-002 used for
X32.c data tables).

**If the SDK won't compile**: fall back to parsing the Link multicast gossip
protocol directly (Link uses UDP multicast 224.76.78.75:20808). Document the
blocker in Progress and open a new task for the fallback approach.
