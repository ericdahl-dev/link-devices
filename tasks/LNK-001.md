## Goal

Create the `esp32/X32Link/` Arduino sketch directory with a minimal WiFi
sketch and a `config.h` that centralises all runtime parameters. Verify the
bare sketch compiles and connects to WiFi before any Link or OSC code is added.

This task targets any ESP32-S3 board (XIAO ESP32-S3, ESP32-S3 Super Mini, etc.).
The first step is to discover the correct FQBN for the board in hand.

## Acceptance Criteria

- [x] Board identified: `arduino-cli board list` run with board plugged in; FQBN recorded in Progress
- [x] Sketch compiles with discovered FQBN without errors or warnings; flash % and DRAM % recorded
- [x] `esp32/X32Link/X32Link.ino` created — WiFi connect, prints IP to Serial
- [x] `esp32/X32Link/config.h` created with all configurable constants (see Notes)
- [x] Upload succeeds; Serial monitor shows assigned WiFi IP within 10 s of RST

## Notes

**Firmware scope**: `esp32/X32Link/` only. Do NOT touch `esp32/X32_emulator/`.

**Step 1 — discover FQBN**:
```bash
arduino-cli board list
```
Look for the ESP32-S3 entry in the output. Common FQBNs for ESP32-S3 boards:

| Board                  | FQBN (starting point)                                                              |
|------------------------|------------------------------------------------------------------------------------|
| XIAO ESP32-S3          | `esp32:esp32:XIAO_ESP32S3:PSRAM=opi,FlashSize=8M,PartitionScheme=default_8MB,USBMode=hwcdc` |
| ESP32-S3 Super Mini    | `esp32:esp32:esp32s3:USBMode=hwcdc,FlashSize=4M,PartitionScheme=default`          |
| Generic ESP32-S3       | `esp32:esp32:esp32s3:USBMode=hwcdc`                                               |

If the board shows `Unknown` FQBN, try:
```bash
arduino-cli board search esp32s3
```
and pick the closest match. The key flags to get right are `USBMode=hwcdc`
(required for USB serial on ESP32-S3) and the correct `FlashSize`.

**Step 2 — compile and record sizes**:
```bash
arduino-cli compile \
  --fqbn <DISCOVERED_FQBN> \
  --verbose \
  esp32/X32Link 2>&1 | grep -E "Sketch uses|Global variables"
```
Record flash % and DRAM % in Progress. This firmware does not need PSRAM —
the Link SDK + WiFi + OSC buffers fit comfortably in internal SRAM even on
boards with no/less PSRAM (unlike the X32 emulator which required 8 MB OPI PSRAM).

**Step 3 — upload**:
```bash
arduino-cli upload \
  --fqbn <DISCOVERED_FQBN> \
  --port /dev/ttyACM0 \
  esp32/X32Link
```

**Serial quirk**: On ESP32-S3 with USB CDC, open the serial port with
DTR=True, RTS=False BEFORE pressing RST. Hardware reset required after upload
(same behaviour as ESP-001 / XIAO ESP32-S3).

**config.h constants** — all values as `#define`:

| Constant              | Type     | Example value       | Notes                          |
|-----------------------|----------|---------------------|--------------------------------|
| `WIFI_SSID`           | string   | `"MyNetwork"`       | 2.4 GHz — ESP32 doesn't do 5G |
| `WIFI_PASS`           | string   | `"hunter2"`         |                                |
| `XR18_IP`             | string   | `"192.168.1.100"`   | Static IP recommended          |
| `XR18_PORT`           | int      | `10024`             | XR18 OSC port                 |
| `LINK_FX_SLOT`        | int      | `1`                 | FX slot 1–4                   |
| `LINK_DEFAULT_BPM`    | float    | `120.0f`            | Initial tempo offered to Link  |
| `LINK_BPM_THRESHOLD`  | float    | `0.5f`              | Min BPM change to trigger OSC  |
| `LINK_POLL_MS`        | int      | `50`                | FreeRTOS task poll interval    |

**Minimal X32Link.ino structure**:
```cpp
#include <WiFi.h>
#include "config.h"

void setup() {
  Serial.begin(115200);
  WiFi.begin(WIFI_SSID, WIFI_PASS);
  while (WiFi.status() != WL_CONNECTED) { delay(500); Serial.print("."); }
  Serial.println();
  Serial.print("IP: "); Serial.println(WiFi.localIP());
}

void loop() { delay(1000); }
```
