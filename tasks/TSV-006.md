## Goal

Implement `esp32/X32ToastSaver/tsv_display.cpp` and `tsv_display.h`: a live touchscreen UI on the Waveshare ESP32-S3-Touch-LCD-4.3 (800×480) using LVGL. Displays the 512-bin power spectrum as a bar chart, marks notched GEQ bands, and shows connection status and active cuts.

## Target board display

The Waveshare ESP32-S3-Touch-LCD-4.3 uses:
- ST7262 RGB LCD driver (16-bit parallel RGB interface, 800×480)
- GT911 capacitive touch controller (I2C)
- LVGL display driver is available in the Waveshare Arduino BSP or the `esp_lcd` panel API

Use the **Waveshare Arduino library** for this board (`Waveshare_ESP32_S3_Touch_LCD_43`) which wraps the display init and LVGL setup. This avoids writing raw `esp_lcd` panel config from scratch.

## Public interface (tsv_display.h)

```cpp
#pragma once
#include <stdint.h>

/* Initialize LVGL and create all UI widgets. Call from setup() last. */
void tsv_display_begin(void);

/* Update the display with current state. Call from loop() after detection.
 * power_db[N/2]: current FFT power spectrum (512 values, dBFS)
 * baseline_db[N/2]: current noise floor (512 values, dBFS)
 * par_active[31]: 1 if GEQ par N is notched, 0 otherwise
 * connected: true if XR18 handshake is active */
void tsv_display_update(const float *power_db, const float *baseline_db,
                         const uint8_t par_active[31], bool connected);

/* Call from loop() — runs LVGL task handler. */
void tsv_display_tick(void);
```

## Implementation guide

### LVGL setup

Use the Waveshare BSP init (or equivalent):
```cpp
#include <Waveshare_ESP32_S3_Touch_LCD_43.h>

void tsv_display_begin(void) {
    /* Board BSP init: sets up RGB LCD + GT911 touch + LVGL */
    Board_Init(BSP_LCD_RESET, BSP_LCD_BL);
    lv_init();
    /* ... BSP display driver registration ... */
    create_ui();
}
```

### Spectrum bar chart

Use an LVGL bar chart (`lv_chart_t`) with 512 data points or a custom canvas draw for performance. Recommendation: **custom canvas draw** — draw vertical bars directly on an `lv_canvas` each frame using `lv_canvas_draw_rect()`. This is faster than managing 512 lv_chart data series.

Bar height: `h = (power_db[i] - baseline_db[i] + 60) / 60 * chart_height` clamped to [0, chart_height].

Bar color:
- Default: `lv_palette_main(LV_PALETTE_GREEN)` (or white)
- Above threshold: `lv_palette_main(LV_PALETTE_YELLOW)`
- Notched band: `lv_palette_main(LV_PALETTE_RED)`

To determine if a bar is notched: map FFT bin → GEQ par via `fft_bin_to_geq_par()` (expose this from `fft_detector.h`), check `par_active[par-1]`.

### Notch marker row

Under the spectrum, draw small downward-pointing triangles (▼) at the FFT bin positions of active notches. Use `lv_canvas_draw_polygon()` or draw a simple filled rectangle.

### Header bar

LVGL label widgets:
- Left: "XAir Toast Saver  ch:N"
- Center: IP address
- Right: connection indicator — green circle `●` if connected, red `○` if not

### Status bar

Two LVGL label widgets updated each frame:
- Line 1: list of active notches ("Notched: 125Hz (-6dB, 3.2s)  1kHz (-6dB, 0.8s)") or "No active notches"
- Line 2: "Threshold: NdB  Cut: NdB  Release: Ns" (static after config load)

### Touch: clear notches

Register a touch callback on the spectrum area: a tap clears all active notches (calls `teq_notch_restore_all()`). This lets the sound engineer manually reset if needed.

## Acceptance Criteria

- [ ] `tsv_display.h` defines the three-function interface above.
- [ ] `tsv_display.cpp` compiles with the Waveshare BSP and LVGL.
- [ ] `tsv_display_begin()` called last in `setup()`; display shows header and empty spectrum on boot.
- [ ] `tsv_display_update()` called each loop iteration; spectrum bars update live.
- [ ] Notched bands render in red; notch markers (▼) appear below the spectrum.
- [ ] Connection status updates: green when XR18 is reachable, red during WiFi loss.
- [ ] Tap on spectrum area calls `teq_notch_restore_all()` and clears all red markers.
- [ ] `tsv_display_tick()` is called from `loop()` — `lv_timer_handler()` runs without starvation.

## Notes

- LVGL `lv_timer_handler()` must be called every ~5ms to avoid display lag. If the main loop blocks in `mic_capture_read()` (which blocks for 1024/48000 ≈ 21ms), call `tsv_display_tick()` after each read — the 21ms block is within LVGL's tolerance.
- The canvas-based approach avoids `lv_chart` performance issues with many data points. Allocate the canvas buffer in PSRAM (`heap_caps_malloc(w*h*2, MALLOC_CAP_SPIRAM)`).
- Expose `fft_bin_to_geq_par()` in `fft_detector.h` so `tsv_display.cpp` can use it for bar coloring.
- Update rate: once per mic frame (~21ms = ~47 fps). LVGL can handle this; the bottleneck is the canvas fill time. If performance is a problem, reduce update rate to every 2nd frame.
- The touchscreen GT911 is I2C; the BSP handles the interrupt and reports events to LVGL's input driver automatically.
