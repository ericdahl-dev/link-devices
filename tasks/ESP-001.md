## Goal

Confirm XIAO ESP32-S3 board, Arduino toolchain, PSRAM, and serial monitor all work before X32 porting begins.

## Acceptance Criteria

- [x] arduino-cli installed; ESP32 core esp32:esp32@3.3.10 installed
- [x] Board on /dev/ttyACM0 (Espressif USB JTAG/serial debug unit, 303a:1001)
- [x] Compiles with FQBN: esp32:esp32:XIAO_ESP32S3:PSRAM=opi,FlashSize=8M,PartitionScheme=default_8MB,USBMode=hwcdc
- [x] Upload succeeds at 921600 baud
- [x] PSRAM: 8192 KB total, 8189 KB free
- [x] PSRAM 1MB read/write: PASS
- [x] CPU: 240 MHz dual-core, Flash: 8 MB

## Notes

**Board**: Seeed Studio XIAO ESP32-S3 (non-Sense; no SD card — Sense addon absent)
**Chip**: ESP32-S3R8, OPI PSRAM 8MB (AP_3v3)

**USB quirk**: Auto-reset after upload leaves board in bootloader. Must press physical RST to run sketch.

**Serial monitor**: Open /dev/ttyACM0 with DTR=True, RTS=False BEFORE pressing RST.
```python
s = serial.Serial('/dev/ttyACM0', 115200, timeout=0.3, xonxoff=False, rtscts=False, dsrdtr=False)
s.dtr = True; s.rts = False
# press RST now
```

**State storage**: LittleFS in flash (SD card deferred).

**Upload command**:
```
arduino-cli upload \
  --fqbn esp32:esp32:XIAO_ESP32S3:PSRAM=opi,FlashSize=8M,PartitionScheme=default_8MB,USBMode=hwcdc \
  --port /dev/ttyACM0 <sketch_dir>
```
