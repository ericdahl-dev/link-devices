## Problem (architecture review candidate C)

Understanding "how Link works" means holding **6 files across 2 dirs** in your
head: `link_protocol.c` (parse) → `wifi_link.c` (listener) → `link_measure_io.c`
(ping/pong) → `link_measurement.c` (TLV+GhostXForm) → `link_measurement_session.c`
(measure policy/epoch) → `link_phase.c` (host→ghost + beats + epoch detector).
The split isn't clean by concept:
- **GhostXForm is split across two headers** — `LinkGhostXForm` +
  `host_to_ghost` in `link_measurement.h`, but the consumer `link_phase_beats_now`
  in `link_phase.h`.
- **Epoch-reset is authored in one module, enforced in another** — detector
  `link_phase_timeline_epoch_reset` in `link_phase.c`, only caller in
  `link_measurement_session.c`. AGENTS.md needs a paragraph to explain the split.

## Solution

Group the four **pure** Link modules (`link_protocol`, `link_measurement`,
`link_measurement_session`, `link_phase`) into a `link/` subdir with one "Link
engine" header/README stating the pipeline once (gossip → measurement →
ghost-xform → beats/phase → epoch-reset). Keep the two socket files
(`wifi_link`, `link_measure_io`) as platform glue outside it.

## Deliverable

`link/` holds the pure engine + its tests + a pipeline README; includes/CMake/
symlinks updated. Pure relocation — no logic change, tests unchanged. Host suite +
both firmware builds green.

## Notes

- Coordinate with ARC-009 (shared/ hoist) — the `link/` dir may live under the
  neutral shared root rather than X32Link/.

## Decision (2026-07-05): declined — see ADR-0006

X32Link builds with **arduino-cli**, which compiles only the sketch **root** (+
`src/`), not arbitrary subdirs. Moving the four Link modules into `link/` stops
them compiling into the S3 firmware unless each is symlinked back into the sketch
root (the pattern X32MidiClock already uses for ~16 shared files). The
navigability gain doesn't justify a symlink farm in front of flashed firmware.
The high-value part — a documented pipeline — is served by header comments +
`AGENTS.md` instead. Reopen only if X32Link migrates off arduino-cli.
