## Goal

Extract the free-running beat generator (`60000/bpm` interval + edge-detect +
self-clear) that `tempo_source_beat()` implements inline against `millis()` into a
pure, host-tested `beat_synth` module — the same idea already lives in
`clock_ticker.c` and is re-derived a third time in the web `setBeat()` JS.

## Problem

- `tempo_source.cpp` `tempo_source_beat()` holds a stateful `millis()`-timed
  interval generator in untestable glue. The `led_task` fallback path
  (`X32Link.ino`) and the web `setBeat()` JS re-derive the same `60000/bpm`
  free-run blink — three copies of one idea, none host-tested on-device.
- Separately, `tempo_source` is accreting: the Link "current position" preamble
  (`link_proto_timeline` + `link_measurement_current_xform` +
  `link_ghost_xform_host_to_ghost(esp_timer_get_time())`) is inlined in
  `tempo_source_phase()`, `tempo_source_beats_now()`, and `tempo_source_phase_valid()`.

## Design (not binding)

- Pure `beat_synth.{h,c}`: `bool beat_synth_step(BeatSynth* s, uint32_t now_ms,
  float bpm)` — true once per `60000/bpm` interval; resets when `bpm <= 0`. Owns
  the interval state.
- `tempo_source_beat()` calls it (glue passes `millis()` + `link_listener_bpm()`).
- Optional (Worth-exploring half): a private `link_position_now(&beats, &valid)`
  helper so `phase()`/`beats_now()`/`phase_valid()` each become one line over it.

## Acceptance Criteria

- [ ] `beat_synth.{h,c}` pure, wired into `esp32/test/Makefile` with
      `test_beat_synth.c`: fires once per interval, no double-fire within an
      interval, resets/returns false when `bpm <= 0`, handles bpm changes.
- [ ] `tempo_source_beat()` delegates to `beat_synth`; no behavior change.
- [ ] (If done) the Link-position preamble is factored to one helper.
- [ ] Full host suite green; firmware compiles; `AGENTS.md` module table updated.
