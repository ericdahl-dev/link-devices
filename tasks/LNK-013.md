## Goal

Add a shared **OSC receive** path to the X32Link module set. Today the firmware
only *sends* OSC (`osc_out.*` / `osc_sender.*`); the new control firmwares
(FaderDisp `FDR-`, SafeMutes `MUT-`, MidiOscIttt `ITT-`) all need to **subscribe
to mixer changes and parse incoming OSC messages**.

This task creates `osc_in.{h,c}` (pure parse) + `osc_listener.{h,cpp}` (the
Arduino UDP + keepalive glue), living in `esp32/X32Link/` and symlinked into the
consuming sketches — the same real-file/symlink pattern as `osc_out` /
`osc_sender` (see `esp32/AGENTS.md`).

## Acceptance Criteria

- [ ] `esp32/X32Link/osc_in.c` + `osc_in.h`: pure C OSC parser. Given a UDP
      payload, exposes the address string, the type-tag string, and decoded
      args (int32, float32, string) via a small struct. **No networking** here —
      host-testable.
- [ ] `esp32/X32Link/osc_listener.cpp` + `.h`: opens a UDP socket, sends
      `/xinfo` handshake, sends `/xremote` on start and **re-sends every ≤9 s**,
      receives datagrams in the FreeRTOS loop, and invokes a registered
      `void (*on_message)(const osc_msg_t*)` callback per parsed message.
- [ ] Big-endian decode correct: int32/float32 read MSB-first; strings
      NUL-terminated and 4-byte aligned (verify against `osc_out.c` encoder).
- [ ] Host test `esp32/test/test_osc_in.c` (Unity): feeds canned packets
      (`/ch/01/mix/fader ,f 0.75`, `/ch/03/mix/on ,i 0`) and asserts the parsed
      address/tags/args. Wired into `esp32/test/Makefile`.
- [ ] Integration check: against the on-device **X32 emulator**
      (`esp32/X32_emulator/`), moving a fader in a connected client (X32-Edit /
      Mixing Station) causes `osc_listener` to log the matching
      `/ch/NN/mix/fader` message. Record in Progress.

## Notes

- Reuse, don't fork: `osc_out.c` already builds packets and knows the
  `config_model_port()` 10023/10024 split — share helpers where sensible.
- The emulator keeps an xremote client updated for `XREMOTE_TIME` (11 s); our
  9 s re-send stays inside that window.
- Keep `osc_in.c` allocation-free (caller-provided buffer) — it runs per packet.
- Subscription / keepalive protocol is documented in
  [`docs/xr18-xair-osc-cheatsheet.md`](../../docs/xr18-xair-osc-cheatsheet.md)
  ("Transport") and [`docs/xr18-meters-osc.md`](../../docs/xr18-meters-osc.md).
- No AppConfig changes needed; `mixer_ip` + model/port already exist.

## Progress (2026-06-29) — module delivered

- `esp32/X32Link/osc_in.{c,h}`: pure, allocation-free OSC parser → `osc_msg_t`
  (address, tag string, decoded int32/float32/string args; big-endian; unknown
  tags stop gracefully). **8 host tests** (`test_osc_in.c`): float/int/string,
  multi-arg, empty tags, unknown-tag stop, non-address + truncated rejection.
  Encoding mirrors `osc_out.c` (same OSC layout).
- `esp32/X32Link/osc_listener.{cpp,h}`: WiFiUDP glue — `/xinfo` + `/xremote` on
  begin, re-send every 8 s (inside the 11 s window), recv loop → `osc_in_parse`
  → registered `on_message` callback. Compiles into X32Link (35%).
