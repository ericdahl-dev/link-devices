## Goal

Add **swing/shuffle** per clock output (the last E-RM-Multiclock-style control),
on top of the per-output division + phase-nudge shipped in P4-010: delay the
off-beat subdivisions so a straight clock grooves (e.g. 54–66% swing).

## Why it's separate from P4-010

`clock_ticker` / `clock_output` are **count-based** — "how many pulses are due
since the last call" — which cleanly models division + a linear phase shift but
**not swing**: swing is a non-linear time-warp that delays specific subdivisions
(the "and" of each beat), so it needs per-pulse *timing*, not a count. Bolting it
onto the count model would be wrong; it wants its own small design.

## Scope

- Pure swing module (TDD'd): given a linear beat position + swing amount +
  subdivision, return the warped position (push the off-beat later). Compose with
  `clock_output` so each output warps before quantizing.
- A `clk<N>_swing` config field (0 = straight) + web-UI control (P4-010's row).
- Verify by ear / MIDI monitor: a swung output grooves; a 0-swing output is straight.

## Notes

- Pairs with P4-010's per-output config surface (add one field + control).
