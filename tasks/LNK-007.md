## Goal

Serve a browser-based configuration page from the ESP32 so end users never
touch the source code. Settings are persisted to NVS and loaded at boot.
This is the only place users configure the device.

## User-Facing Settings

| Field            | Options / default                         |
|------------------|-------------------------------------------|
| Mixer model      | XR18 / XR16 / XR12 ¬∑ X32 / X32 Compact  |
| Mixer IP address | 192.168.4.1 (default ‚Äî XIAO SoftAP)      |
| FX slot          | 1‚Äì4 (XR18) or 1‚Äì8 (X32)                  |
| WiFi SSID        | text field                                |
| WiFi password    | password field                            |

Mixer model auto-sets the OSC port (XR18‚Üí10024, X32‚Üí10023). Users never
see or type a port number.

## Acceptance Criteria

- [ ] HTTP server starts after WiFi connects; accessible at `http://{ESP32-IP}/`
- [ ] Config page renders correctly on mobile and desktop
- [ ] Submitting the form saves all fields to NVS and redirects back with
      a "Saved ‚Äî restarting‚Ä¶" message, then reboots the ESP32
- [ ] On boot, firmware reads config from NVS; falls back to hardcoded
      defaults if NVS is empty (first-run or after factory reset)
- [ ] Changing mixer model dynamically updates the FX slot range shown in
      the UI (JS: 4 slots for XR18, 8 for X32)
- [ ] WiFi credentials changeable via the UI; reconnect on reboot
- [ ] Factory reset: holding BOOT button for 5 s clears NVS and reboots
      into default config

## Architecture

**Storage**: `Preferences` library (NVS key-value, built into arduino-esp32).
No filesystem required. Keys: `model`, `mixer_ip`, `fx_slot`, `wifi_ssid`,
`wifi_pass`.

**Web server**: `WebServer` library (built into arduino-esp32). Single-page
form served from a `const char*` in PROGMEM. No external files or SPIFFS.

**Access**: User browses to the ESP32's IP address (shown on Serial at boot)
while connected to the same WiFi network as the mixer.

**Runtime**: Config values replace compile-time constants (`MIXER_IP`,
`MIXER_PORT`, `LINK_FX_SLOT`) everywhere in LNK-005 OSC output code.

## Notes

- `config.h` retains only `LINK_DEFAULT_BPM`, `LINK_BPM_THRESHOLD`,
  `LINK_POLL_MS` ‚Äî values that are not user-facing. All mixer/slot/WiFi
  settings move to NVS.
- The web server runs concurrently with the BPM poll task. Keep the HTML
  payload small (< 4 KB) to avoid heap pressure. Inline CSS only, no JS
  frameworks.
- LNK-005 must read `g_config.mixer_ip`, `g_config.mixer_port`,
  `g_config.fx_slot` from a globally accessible config struct populated
  at boot, not from `#define` constants.
