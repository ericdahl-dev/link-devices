## Goal

One declared interface for "where does the BPM come from", with BOTH inputs
compiled into a single firmware and the active one chosen at **runtime**. This
is the foundation for collapsing X32Link + X32MidiClock into one firmware
(LNK-010) configured from the web UI (LNK-012). Today the two firmwares are
the same application with two input modules and no interface between them.

## Proposed interface

New `tempo_source.h` (C linkage). Both adapters compile in; `tempo_source.c`
dispatches to the selected one:

```c
#pragma once
#include <stdint.h>
#include <stdbool.h>

typedef enum { TEMPO_SRC_LINK = 0, TEMPO_SRC_MIDI = 1 } TempoSourceKind;

// Choose the active input. Call once at boot before pre_net/begin.
void  tempo_source_select(TempoSourceKind kind);

void  tempo_source_pre_net(void);  // pre-WiFi init (USB MIDI enumerates here)
void  tempo_source_begin(void);    // post-WiFi init (Link joins multicast here)
void  tempo_source_poll(void);     // service input every bpm_task tick
float tempo_source_bpm(void);      // current BPM; 0.0 = no live signal
bool  tempo_source_active(void);   // peers>0 / pulses within timeout
bool  tempo_source_beat(void);     // true once per beat, self-clears (LED)
```

**Dispatch design:** `tempo_source.c` holds the selected `kind` and forwards
each call to namespaced adapter functions (`link_tempo_*`, `midi_tempo_*`).
Both adapters are always linked; selection is a runtime branch / function
table. No symbol collisions — adapters keep distinct prefixes.

**Two-phase begin** captures the opposite ordering constraint: USB MIDI must
enumerate BEFORE WiFi; Link joins multicast AFTER. `pre_net` always runs the
MIDI init (harmless in Link mode); `begin` runs the Link join.

Adapters (per-input `.cpp`, not symlinked):
- `tempo_source_link.cpp` — `pre_net`=noop; `begin`=`link_listener_begin`;
  `poll`=`link_listener_poll`+`link_listener_tick`; `bpm`=`link_listener_bpm`;
  `active`=`link_listener_peers()>0`; `beat`=synthesise from bpm
  (mirror `led_task` math: `beat_ms = 60000/bpm`).
- `tempo_source_midi.cpp` — `pre_net`=`midi_clock_init` (+ the `USB.*` calls
  from the .ino); `begin`=noop; `poll`=noop (task-driven);
  `bpm`=`midi_bpm_update`; `active`=`bpm>0`; `beat`=`midi_clock_beat_flag`.

## Constraints

- **Arduino flat sketch**: shared files via symlink. `tempo_source.h` + the
  dispatcher `tempo_source.c` are shared (symlinked); the two adapter `.cpp`
  files are real and live in the sketch dir.
- Both adapters must compile together in one TU set (no duplicate global
  symbols, no clashing static state names).
- Keep `link_protocol.c` and host tests untouched and green.
