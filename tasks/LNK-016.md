## Goal

Validate the LNK-014/015 touch UI on a real Waveshare ESP32-S3-Touch-LCD-1.47
unit against a real XR18 or X32, the same way LNK-006 validated the original
firmware against real hardware. Confirm the touchscreen is a fully usable
replacement for the web config flow, not just a UI that compiles.

## Test plan

1. **Cold boot, factory config**: flash a unit with NVS cleared. Confirm the
   status screen shows "no signal" / disconnected state cleanly (no garbage
   readout) before any config is set.
2. **Touch-only setup**: with no phone/laptop on the network, configure
   WiFi SSID/password, mixer IP, mixer model, FX slot, and tempo source
   entirely via the touchscreen. Confirm "Write & Reboot" persists and the
   unit reconnects on the configured network after reboot.
3. **Live tempo accuracy**: start an Ableton Link session (or USB MIDI clock,
   per configured source) at a known BPM. Confirm the status screen's BPM
   readout and beat blink match the source within the existing
   `LINK_BPM_THRESHOLD` / `MCK_BPM_THRESHOLD` tolerances, and that OSC
   reaches the mixer's FX delay parameter (`/fx/{slot}/par/01`) correctly —
   same check as LNK-006, now confirmed against the touch-configured values.
   Also confirm the phase wheel (LNK-015/LNK-019) shows "syncing…" + beat
   blink immediately after boot, then switches cleanly to the real phase
   sweep once `tempo_source_phase_valid()` goes true — this is the
   touchscreen-specific check; `LNK-020` covers phase/LED accuracy in depth,
   this step only needs to confirm the on-screen widget itself behaves.
4. **Concurrent web UI**: with the touch-configured unit running, open the
   web config page from a browser. Confirm it shows the same live BPM and
   that changing a setting from the web form updates the touch status screen
   after reboot (and vice versa) — both front ends agree, neither stomps
   the other's NVS write.
5. **Stability under load**: run for at least 30 minutes with both
   `web_config_handle()` and `touch_display_tick()`/`touch_display_update()`
   active. Watch `ESP.getFreeHeap()` (already logged every 5s in `loop()`)
   for downward drift — no leaks from LVGL screen swapping or repeated
   keyboard widget creation/teardown.
6. **Factory reset**: confirm holding BOOT for 5s still clears NVS and
   reboots to default config, with the touch UI reflecting cleared defaults
   on the settings screen afterward.

## Acceptance Criteria

- [x] Steps 1-4 pass on real hardware (touch-only setup, live tempo + phase
      wheel, concurrent web UI). Steps 5 (30-min heap soak) and 6 (BOOT 5s
      factory reset) not formally run this pass — see Progress.
- [ ] No heap drift over a 30-minute soak (free heap within noise of its
      starting value, not trending down). **Not run** — deferred (low risk: the
      UI is raw LovyanGFX, not LVGL, so the `lv_keyboard`-recreation leak this
      criterion guarded against does not exist).
- [x] Touch settings screen and web config page never disagree about the
      currently-saved configuration after either one writes.
- [x] Results recorded in this file's Progress section, including which
      touch controller chip (AXS5106L, per LNK-014) was confirmed
      present on the validated unit.

## Notes

- This is a hardware-in-the-loop task, like LNK-006 — host tests (`esp32/test`)
  don't cover LVGL/display/touch behavior. Run it on the physical board.
- If step 5 shows heap drift, suspect repeated `lv_keyboard` create/delete on
  every settings-screen entry — consider creating it once and
  showing/hiding rather than recreating per visit.
