## Goal

Make the metronome (P4-006) **volume** and **click sound** user-configurable from
the web UI, instead of the compile-time constants they are today.

## What's fixed today (all in `P4Hub/main/metronome_audio.c`)

- **Volume:** `es8311_voice_volume_set(codec, VOLUME=80, …)` — the ES8311 DAC
  volume is a 0–100 value, hardcoded.
- **Click sound:** synthesized tone bursts — `CLICK_HZ 1000`, `ACCENT_HZ 1760`,
  `CLICK_MS 45` / `ACCENT_MS 55`, `ACCENT_AMP 0.55`, sine + 4 ms/8 ms
  attack/release envelope. All `#define`s.

## Scope

- **Volume:** a 0–100 config field (`metronome_volume`) → `es8311_voice_volume_set`.
  Web UI slider (or the `.fld` number field). NVS-persisted.
- **Click sound:** expose the tone parameters — at minimum click/accent **pitch**
  and **length**; optionally a small set of named **voices** (e.g. Tone / Click /
  Woodblock) as presets that map to those params. Web UI select + fields.
- Refactor `metronome_audio.c` so the burst is (re-)rendered from **runtime**
  params (a `metronome_audio_config(...)` setter) instead of `#define`s; re-render
  the buffers when config changes.

## Architecture (ADR-0003 / 0004)

- Config lives in the **web UI** (ADR-0004). Add fields to the pure
  `p4hub_config.c` (`metronome_volume`, click params / voice enum) with range
  validation, extend `test_p4hub_config.c` (red-green). The codec/I2S rendering
  stays glue in `metronome_audio.c`.
- Keep the pure `metronome.c` click/accent *scheduler* unchanged — this ticket is
  only about how a click *sounds*, not when it fires.

## Deliverable

Metronome volume + click sound adjustable and saved from the web UI; verified by
ear on the P4-NANO speaker.
