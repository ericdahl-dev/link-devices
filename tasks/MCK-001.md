## Goal

Create the `X32MidiClock/` Arduino sketch directory for the USB-MIDI-clock-to-X32
firmware. This is a separate device from X32Link: the musician plugs a USB cable
into their laptop, Ableton sees a class-compliant MIDI device, sends clock, and
the ESP32-S3 forwards tempo to the X32/XR18 via OSC over its own WiFi connection.

## Acceptance Criteria

- [ ] `X32MidiClock/X32MidiClock.ino` exists (stub that compiles — `setup()`/`loop()` only)
- [ ] `X32MidiClock/build_opt.h` sets USB MIDI mode flags for ESP32-S3 (see Notes)
- [ ] `X32MidiClock/config.h` with MCK-specific timing constants (no Link references)
- [ ] Shared modules symlinked from `X32Link/` into `X32MidiClock/`:
  - `app_config.h / app_config.c / app_config_nvs.cpp`
  - `osc_out.h / osc_out.c`
  - `osc_sender.h / osc_sender.cpp`
  - `bpm_tracker.h / bpm_tracker.c`
  - `web_config.h / web_config.cpp`
- [ ] Stub compiles clean in Arduino IDE with board = ESP32-S3 Dev Module, USB Mode = USB-OTG

## Notes

**Why symlinks**: Arduino requires all `.cpp`/`.c`/`.h` files to be inside the
sketch directory. Symlinks let both firmwares share the same source without
duplicating or restructuring into a library. `git` tracks symlinks fine.

**build_opt.h** for USB MIDI on ESP32-S3:
```
-DARDUINO_USB_MODE=1
-DARDUINO_USB_CDC_ON_BOOT=0
```
USB Mode must be set to "USB-OTG" in Arduino IDE board settings (not "UART0").
When USB-OTG is active, `Serial` is unavailable — use `Serial0` for UART debug
or omit debug output entirely.

**config.h** contents (MCK-specific, no Link):
```c
#pragma once
#define MCK_DEFAULT_BPM       120.0f
#define MCK_BPM_THRESHOLD     0.5f
#define MCK_CLOCK_WINDOW      24     // pulses to average (1 beat = 24 PPQN)
#define MCK_CLOCK_TIMEOUT_MS  2000   // stop sending if no clock for this long
```
