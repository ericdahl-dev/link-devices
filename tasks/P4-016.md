## Goal

Stand up a **headless, always-on ground-truth Ableton Link peer** on the Arduino
Uno Q's Linux side, running the **official Ableton Link SDK**, to verify the
ESP32 devices' Link behaviour against a real peer — not against our own
clean-room implementation.

## Why this exists

The ESP32 firmware speaks Link via a clean-room reimplementation (no official
SDK — it can't run on a microcontroller). That's fine for *listening*, but the
hard, unverified work is the P4 **publishing** into a session:

- **P4-011 stage 2** — the P4 becomes a tempo-setting peer. Its correctness ("do
  other peers actually adopt the P4's tempo/timeline?") **cannot be checked by CI
  or host tests** — it needs a real Link peer watching the session.
- **P4-009** phase-lock and tempo-in accuracy also want ground-truth comparison.

Today this is done ad-hoc against the Ableton Link SDK on a Mac (the reference-
probe rig). The Uno Q can be a **dedicated, headless, always-on** version of that
rig that lives on the bench.

## Why the Uno Q specifically

It is a **Linux board** (Qualcomm Dragonwing quad Cortex-A53 running Debian +
STM32U585 MCU + WiFi), so unlike the ESP32 it can run the **official Link C++
SDK** directly. It does **not** run this firmware and is **not** a product tier —
it's a bench instrument. (A separate someday-idea, out of scope here: a full Link
hub that runs the official SDK on Linux and does tight MIDI clock on the STM32
brain.)

## Scope

- Build/run the official Link SDK (e.g. `LinkHut` or a small custom peer) on the
  Uno Q Debian side; join the same WiFi/session as the P4.
- Log what a real peer observes: session **tempo**, **beat/phase**, **peer
  count**, and transport (start/stop) — timestamped, so a run can be diffed
  against what the P4 *thinks* it published.
- A simple pass/fail check for stage-2: with the P4 set as MIDI-in-driven master,
  confirm the Uno Q peer adopts the P4's tempo (and the beat stays continuous on
  takeover), then re-confirm when a DAW peer overrides.
- Document the setup so it's repeatable (network, SDK build steps, how to read the
  log). Mirror/replace the Mac reference-probe-rig notes.

## Deliverable

A documented, repeatable Link reference peer on the Uno Q that reports live
session tempo/phase/peers/transport, used to verify P4-011 stage 2 (and phase-
lock) against ground truth instead of self-comparison.

## Notes

- Pure bench tooling — no change to the ESP32 firmware or the shared pure modules.
- Complements, doesn't replace, the host-tested pure logic; it covers exactly the
  on-the-wire behaviour that host tests can't.
