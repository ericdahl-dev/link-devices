## Goal

Pull the measurement-session *policy* out of `link_measurement_io.cpp` (thin
Arduino glue, per ADR-0003) into a **pure, host-tested** state machine. Today
~120 of the file's ~160 lines are branching orchestration logic living in glue
that "isn't meant to be tested" — and **both LNK-026 bugs lived there**, catchable
only by flashing hardware.

## Problem

`link_measurement_io.cpp` claims (header) to be thin glue, but owns:
- `check_trigger()` — peer selection + `different_peer` detection + `due`
  re-measure timer + gating on `link_measurement_active()`.
- `check_epoch_reset()` — origin-jump detection + reset/re-measure response
  (LNK-026 bug 1).
- RX flush in `start_attempt()` (LNK-026 bug 2).
- The 50 ms watchdog / 5-consecutive-timeout retry state machine in `_poll()`.
- 8 `static` state vars (`s_have_ref`, `s_ref_ip/port`, `s_have_prev_ghost`,
  `s_prev_ghost_us`, `s_last_send_us`, `s_consecutive_timeouts`,
  `s_next_measure_us`, `s_have_last_origin`, `s_last_time_origin_us`).

`link_measurement.c` (the "pure" module) only owns the sample buffer + committed
xform — the bug-prone *sequencing* is all in glue.

## Design (not binding)

A pure `link_measurement_session.c`:
- `void session_reset(Session*)`
- `int session_poll(Session*, const SessionInputs* in, SessionAction* out, int max)`
  returning an ordered action list.
- **Inputs** (facts the glue already gathers): `now_us`; peer `{found, ip, port}`;
  timeline `{valid, time_origin_us}`; `samples_count`; `active`; and a pong event
  `{arrived, has_ghost, ghost_us, host_echo_us}`.
- **Actions**: `FLUSH_RX`, `START_ATTEMPT(ip,port)`, `SEND_PING(prev_ghost?)`,
  `END_ATTEMPT(success)`, `RESET_XFORM`.

The glue (`link_measurement_io.cpp`) becomes a mechanical translator: read
peer/timeline/socket facts → `session_poll` → execute the actions (sockets only).
The existing pure `link_phase_timeline_epoch_reset()` and the sample/xform funcs
in `link_measurement.c` stay as-is and are called by the session or the glue.

## Acceptance Criteria

- [ ] `link_measurement_session.{h,c}` pure (no Arduino headers), wired into
      `esp32/test/Makefile` with `test_link_measurement_session.c`.
- [ ] Host tests cover the LNK-026 regressions as unit tests: epoch-reset drops
      the xform + re-measures the same poll; a stale pong across the idle gap is
      flushed at attempt start; watchdog fires at 50 ms; 5 timeouts abandon
      without clobbering the committed xform; different-peer and due-timer both
      retrigger; prime/prev-ghost carry-over across rounds.
- [ ] `link_measurement_io.cpp` reduced to socket + timing glue that drives the
      session; no behavior change vs today (LNK-026 fixes preserved).
- [ ] Full host suite green; firmware compiles; `esp32/AGENTS.md` updated.
- [ ] Re-verified on hardware against a Link peer (steady play + a stop/start)
      that phase still holds and re-locks — no LNK-026 regression.
