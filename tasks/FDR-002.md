## Goal

Implement the FaderDisp behaviour: when a channel fader moves, show its value in
**dB on that channel's scribble strip**, by writing `/ch/NN/config/name`. On a
clean shutdown / disable, restore the original names.

## Acceptance Criteria

- [ ] `fader_disp.{cpp,h}` added to `esp32/X32FaderDisp/`.
- [ ] On each received `/ch/NN/mix/fader ,f <f>`, convert `f`→dB with the
      **exact** piecewise formula in
      [`docs/xr18-xair-osc-cheatsheet.md`](../../docs/xr18-xair-osc-cheatsheet.md)
      ("Fader / level: float ↔ dB"), and send
      `/ch/NN/config/name ,s "<dB>"` (e.g. `"-6.0"`, `"+10"`, `"-oo"`).
- [ ] The float→dB conversion is a **pure host-tested** function
      (`fader_to_db()` in a `.c` or testable unit); Unity test asserts boundary
      cases: `f=1.0→+10`, `f=0.75→0`, `f=0.5→-10`, `f=0.25→-30`, `f=0→-oo`.
- [ ] Per-channel throttle of `FDR_NAME_REFRESH_MS` so a fast fader sweep does
      not flood the mixer.
- [ ] **Name save/restore:** on first sight of a channel, cache its original
      `/ch/NN/config/name`; on disable (web toggle) or graceful stop, write the
      cached names back. Guard with `FDR_RESTORE_ON_EXIT`.
- [ ] Web UI exposes an on/off toggle and channel-count (16/32) selector; value
      persists in NVS via AppConfig.
- [ ] Demoed against the on-device X32 emulator: moving a fader updates the
      corresponding channel `name`; a second client (X32-Edit / Mixing Station)
      shows the dB text. Record in Progress.

## Notes

- Writing `config/name` is **destructive** to user labels — that's why
  save/restore is mandatory, not optional. Cache must survive the session in
  RAM (and ideally LittleFS) so a crash doesn't strand the mixer showing dB
  numbers as names.
- dB string format: 1 decimal for ≥ -10 dB region is fine; clamp to `-oo` at
  the bottom. Keep ≤ 7 chars (scribble width).
- Subscribe to the full `/ch/*/mix/fader` space via `/xremote` (the mixer pushes
  all changes; filter by address prefix in the callback).
