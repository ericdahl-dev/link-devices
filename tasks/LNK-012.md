## Goal

Let the user choose the tempo input — Ableton Link or USB MIDI clock — from the
existing web config page, persisted to NVS and applied at boot. This is what
makes the single firmware (LNK-010) a true one-flash device: pick the source in
the browser, no re-flash.

## Current state

`AppConfig` (`app_config.h`) holds `wifi_ssid/wifi_pass/mixer_ip/model/fx_slot`,
persisted via `app_config_nvs.cpp` (NVS `Preferences`) and edited through
`web_config.cpp` (single-page form served from PROGMEM, see LNK-007). There is
no input-source field yet.

## Proposed change

1. **AppConfig**: add `int input_source;` (0 = Link, 1 = USB MIDI), matching the
   `TempoSourceKind` enum from LNK-008. Default = Link.
2. **NVS** (`app_config_nvs.cpp`): load/save the new key `input_src`; default to
   Link when absent (first boot / post factory-reset).
3. **Web form** (`web_config.cpp`): add a labelled dropdown
   "Tempo source: Ableton Link / USB MIDI clock"; submit persists it like the
   other fields, then reboots (existing save→restart flow).
4. **Boot** (LNK-010 `setup`): `tempo_source_select(cfg.input_source)` before
   `tempo_source_pre_net()`.
5. **Validation** (`app_config.c::config_validate`): accept only 0/1.

## Constraints

- USB-MIDI is always compiled in (LNK-010), so switching to MIDI needs no
  re-flash — only the NVS field + reboot.
- Keep the form mobile/desktop friendly and the save→reboot UX from LNK-007.
- Adding a field must not break existing NVS reads (absent key → Link default).

## Acceptance Criteria

- [ ] `AppConfig.input_source` added; `config_validate` accepts only {0,1}.
- [ ] NVS load/save round-trips the field; absent key defaults to Link.
- [ ] Web form shows a "Tempo source" dropdown reflecting the saved value;
      submitting persists and reboots.
- [ ] After reboot, the firmware runs the selected input (verified on QtPy:
      set Link → joins session; set MIDI → USB clock drives BPM), no re-flash.
- [ ] Factory reset (BOOT 5 s) clears the field back to Link default.
- [ ] Host tests still pass: `cd test && make` (extend `test_app_config.c` for
      the new field's validation + default).
