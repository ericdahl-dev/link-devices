## Goal

A visual metronome on an external addressable LED strip, with its own on/off
switch independent of the audio metronome.

## Hardware finding

The P4-NANO's onboard "USER-LED" is a **power indicator** (LED1: ESP_3V3 -> R10 ->
LED -> GND), not on a GPIO — confirmed from the schematic and the absence of any
board-LED example (Waveshare's `05_gpio_io` demo drives user-wired header pins).
So the visual metronome uses an **external strip on a free header GPIO**. Using an
**8-pixel WS2812 strip on GPIO2** (Waveshare's vetted free demo pin), +5V, GND.

## Design — bar-position chase

A lit block walks across the strip, one pass per bar (4/4 assumed, 2 pixels/beat
on 8 pixels): the bar-1 downbeat block is amber, other beats green, dimming across
each beat so it pulses. You see where you are in the bar and the downbeat coming.

## Implementation

- Pure `metro_strip.c` (`metro_strip_render(beats, quantum, npix, out[])`) — the
  chase pattern from the shared beat position; host-tested (`test_metro_strip`,
  6 cases incl. wrap, fade, negative-beat safety). No I/O.
- Glue `ks_led.c` — WS2812 over the RMT `led_strip` managed component
  (`espressif/led_strip`), init/show/clear on GPIO2.
- `clock_out_task` renders the strip from `bs.beats` throttled to ~50 fps
  (decoupled from the 1 ms clock), driven directly by the beat — no coupling to
  the audio click scheduler.
- `led_enable` config field (own switch, default off), validated, `led` key,
  cleared-on-absence for `/save`, and on the P4-015 `/live` path with a web toggle
  ("LED Strip · Visual Metronome") — so it flips with no reboot.

## Status

Built, host-tested, and flashed. Colours kept below full-scale (~0.5 A max at 8
px) for the eyes and the 5 V rail. Needs an on-hardware eyeball: strip chases in
time, amber downbeat, toggles live. 3.3 V data usually drives a short strip fine;
add a level shifter if the first pixels flicker.
