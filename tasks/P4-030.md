## Problem / opportunity

The P4 emits MIDI clock + notes over USB-MIDI today. RTP-MIDI (AppleMIDI) would let
it appear as a **Network MIDI** device to any DAW — macOS/iOS have it natively (Audio
MIDI Setup → Network), Windows via Tobias Erichsen's rtpMIDI — with **no USB cable and
no interface**. RTP-MIDI is MIDI over *IP* (UDP), not Ethernet-specific, so it runs
over the P4's **existing WiFi** (the C6) — this ticket is the WiFi-first bring-up.
Wired Ethernet/PoE is the follow-on [[P4-031]].

Turns the P4 into: Ableton Link in → **Network MIDI out** + USB-MIDI out + (future)
DIN — the KitchenSync hub story ([[p4hub-product-vision]]).

## Scope

Follow the project's pure-engine / thin-glue split (ADR-0003):

- **Pure `rtp_midi_session`** (host-tested, like `link_measurement_session` /
  `session_timeline`): owns the AppleMIDI protocol — the invitation handshake
  (IN/OK/NO on the control port), clock synchronisation (CK0/1/2 timestamps), and
  RTP-MIDI packet framing (RTP header + MIDI command list; the recovery journal can be
  a later refinement — start journal-less, most receivers accept it). No sockets here;
  emit/consume actions like the other session state machines.
- **Thin UDP glue** (`KitchenSync/main/`): two UDP ports (control + data, default 5004
  and 5005), mDNS/Bonjour `_apple-midi._udp` advertisement so it shows up in Audio MIDI
  Setup without manual host:port entry, and wiring the existing MIDI stream (the clock
  task's `0xF8` + note/transport) into the session's send path.
- Reuse the existing 24-PPQN clock + transport source; RTP-MIDI is just another output
  of the same stream (alongside USB-MIDI).

## Notes / risks

- No drop-in ESP-IDF library. lathoub's Arduino-AppleMIDI-Library is the reference
  (Arduino; port the protocol, or reimplement the session pure-C — preferred here).
- **Jitter:** WiFi adds timing jitter, and MIDI clock is timing-sensitive. AppleMIDI's
  timestamped delivery helps, but for the phase-accurate promise the wired path
  ([[P4-031]]) is better — this WiFi bring-up validates the protocol/UX first.
- Validate against macOS Audio MIDI Setup (Network session) and rtpMIDI on Windows.
- Host tests for the handshake state machine + CK sync + packet framing (the interface
  is the test surface).
