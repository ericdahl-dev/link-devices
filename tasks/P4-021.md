## Goal

Emit **analog sync pulses** (0/+5V clock/trigger) on several outputs, each with its
own rate — to sync modular/Eurorack, vintage drum machines (DIN-sync / Sync24),
and pedals, alongside the USB/MIDI clock. Realises the long-planned LNK-028 on the
P4 tier.

## Inspiration

JMK CLOCKstep:MULTI analog sync — a 3.5mm patch bay sends **up to 5x +5V analog
sync** signals, each with a configurable rate (44 rates)
(https://jmkmusicpedals.com/products/clockstep-multi). We can match or exceed the
count.

## Hardware capability (current board)

Analog sync = **digital pulses**, so any free header GPIO is one. After reserving
I2C/I2S/PA/WS2812(GPIO2)/USB/SDIO, the P4-NANO header frees up ~10-16 GPIOs, so
**8-16 analog sync outs** are realistic. A few can use **RMT/MCPWM** for
jitter-free timing; the rest plain GPIO. Two notes:
- Our GPIO is 3.3V; true +5V sync wants a per-output buffer/level-shift (Eurorack
  triggers usually accept 3.3V, but +5V is safer/standard).
- Variable-voltage **CV** (not just sync pulses) is out of scope: the P4 has **no
  onboard DAC** (would need an external DAC chip). Confirm exact free-GPIO / RMT /
  MCPWM channel counts from the P4 datasheet before committing a number.

## Scope

- Reuse the P4-010 pattern: a pure per-output pulse scheduler (division/rate,
  pulse width, phase) driven by the shared beat; host-tested like `clock_output`.
- Glue: drive each output's GPIO (or an RMT/MCPWM channel for the tight ones).
- Web UI: per-output rate/enable rows, same shape as the MIDI clock outputs.

## Deliverable

N analog sync outs, each a configurable rate off the shared Link beat, verified on
a scope / modular. Level-shifting for +5V documented.
