## Source

Design: `docs/plans/2026-07-19-rgb-led-output-subsystem-design.md` (slice 4 of 5). Proves
the option-3 goal: outputs are independent, each assigned a style at runtime.

## What to build

Let the Super Mini's onboard output be switched between **Status** and **Metronome**
(npix=1) at runtime via config — the first output that exercises live style selection.

- The `led[0].style` field is user-settable to STATUS or METRONOME on the Super Mini.
- METRONOME on a 1-px output renders the beat-flash/pattern into a single pixel (FLASH mode
  is the natural default at npix=1) with its configurable beat/accent colors.
- The iOS section shows the style picker; picking Metronome reveals the Metronome fields
  (color/brightness/pattern), picking Status shows only enable + brightness.

## Acceptance criteria

- [ ] `led[0].style` is a live-editable field on the Super Mini; switching Status<->Metronome takes effect with no reboot.
- [ ] With style=METRONOME the onboard LED flashes on the beat in the configured colors; with style=STATUS it shows the fixed state palette — both from the same output.
- [ ] iOS style picker drives the visible field set per style.
- [ ] No regression to the P4 metronome or the Super Mini status behavior when left on their defaults.
- [ ] Host suites green.
