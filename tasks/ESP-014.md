## Goal

Reach the S3 board at `x32link-XXXX.local` instead of hunting for its DHCP
address, the same way KitchenSync does after ESP-012.

## Why

ESP-012's note said to file this "if it lands well". It landed well: the P4
answers at `kitchensync-242c.local` and shows up as `KitchenSync._http._tcp`
in any Bonjour browser. X32Link has the identical problem -- a headless,
OTA-updatable board whose only address discovery is a serial cable.

## Design

Arduino, so `ESPmDNS.h` rather than the IDF component:

    MDNS.begin("x32link-xxxx");   // last two MAC bytes, lowercase hex
    MDNS.addService("http", "tcp", 80);

Two differences from ESP-012 worth knowing before starting:

- `esp_read_mac(ESP_MAC_WIFI_STA)` **works** on the S3 (its WiFi is on-die).
  On the P4 it fails, because the MAC belongs to the C6 -- that cost a
  hardware round-trip in ESP-012. No such trap here.
- `ESPmDNS` has no delegated-hostname API, so the friendly bare alias
  (`x32link.local`) is not free the way it was on the IDF side. Either skip the
  alias or accept the per-unit name only. Skipping is fine: the name is
  discoverable via the `_http._tcp` service anyway.

The name derivation is the same shape as `ks_hostname.{h,c}` (pure,
host-tested). Either generalise that module to take a prefix, or copy it --
ARC-005/ARC-010 would prefer generalising, and it is a two-line change.

## Scope

- `MDNS.begin()` after WiFi connects; `addService("http","tcp",80)`.
- Per-unit name from the MAC; log it next to the IP at boot.
- Advertise in the `X32Link-Config` SoftAP fallback too.

## Non-goals

- Cross-subnet discovery. mDNS is link-local by design (see ESP-012).

## Acceptance Criteria

- [ ] `http://x32link-XXXX.local` serves the config page on a fresh boot
- [ ] Chosen hostname logged at boot alongside the IP
- [ ] Two units on one LAN do not collide
- [ ] Name resolves in SoftAP fallback mode too
- [ ] Name derivation is pure + host-tested (shared with or copied from `ks_hostname`)

## Notes

- Flash cost: the IDF mdns component cost KitchenSync ~48 KB (app partition free
  space went 11% -> 6%). X32Link is at ~81% of a 1.25 MB sketch partition, so
  check `ESPmDNS`'s cost before assuming it fits.
- Pairs with LNK-034 (OTA): `curl -F ... http://x32link.local/update` beats
  looking up an IP first.
