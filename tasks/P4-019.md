## Goal

Make the WS2812 visual metronome (P4-018) customizable from the web UI, live (no
reboot, via the P4-015 `/live` path).

## Controls

- **Brightness** 0..100 % master scale (also caps 5 V current draw).
- **Pattern** — `Chase` / `Flash` / `Fill` dropdown (the three designs).
- **Fade** 0..100 % — how much a beat dims before the next (sharp blink -> smooth pulse).
- **Colors** — beat + accent, via HTML `<input type=color>` (#rrggbb).

## Implementation

- Pure `metro_strip.c` gains a `MetroStripCfg { beat, accent, bright, mode, fade }`
  and implements all three patterns. Host-tested (`test_metro_strip`, 11 cases
  across chase/flash/fill + brightness + fade + custom colors).
- Config fields `led_brightness / led_mode / led_fade / led_beat_color /
  led_accent_color`; `p4hub_config_set` parses `#rrggbb` (or bare hex) for the
  colors. Validated; `test_led_customization`.
- Web UI `build_led()` renders a "Strip Look" panel (brightness/mode/fade + two
  color pickers), all `class="live"`; `/live` copies the fields; the clock task
  builds `MetroStripCfg` from `g_cfg` each frame.

## Status

Built, host-tested, flashed. Colors default green beat / amber downbeat, 60 %
brightness, chase, 55 % fade — matching P4-018's look.
