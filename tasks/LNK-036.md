## Problem

The S3 shows phase as a **sweeping marker orbiting a circle** (a green dot that
travels around the wheel as phase advances). That's one aesthetic. The P4
(KitchenSync) instead **flashes** its LEDs on each beat with distinct colors for
the downbeat vs the other beats — a different, punchier read of the same phase.

Users should be able to pick. Add an option on the S3 to run the phase display
as either the existing **sweep wheel** or a **beat-flash dot** that mimics the
P4's LED behavior, and let the flash colors be configured the same way the P4's
are.

## Fix

Add a display-style choice plus two colors to `AppConfig`, mirroring the P4's
`led_beat_color` / `led_accent_color` (`KitchenSync/main/ks_config.h:37-38`):

- `int phase_display_mode;`  // 0 = sweep wheel (current), 1 = beat-flash dot
- `int dot_beat_color;`      // 0xRRGGBB — beats 2..N (parity w/ P4 led_beat_color)
- `int dot_accent_color;`    // 0xRRGGBB — bar-1 downbeat (parity w/ P4 led_accent_color)

Touched files:
- `X32Link/app_config.h` / `app_config.c` — add fields + defaults (default mode 0
  so existing behavior is unchanged; default colors matching the P4's defaults).
- `X32Link/app_config_nvs.cpp` — load/save the new fields.
- `X32Link/touch_display.cpp` — the render path at `:350-366` (WHEEL_CX/CY/R,
  the `fillCircle(mx,my,6,TFT_GREEN)` sweep marker). In mode 1, instead of the
  orbiting marker, fill the wheel/dot on each beat: **accent color on the
  downbeat (bar-1), beat color on beats 2..N**, using the bar position from the
  same phase math that drives the sweep. Decide fade/hold (a P4-style `led_fade`
  dim-across-the-beat is a nice-to-have; colors are the core ask). Convert
  0xRRGGBB → LovyanGFX `color565`.
- `X32Link/web_config.cpp` — expose mode + two color pickers in the config form
  (see the SSID field pattern at `:152`, save at `:360`).
- Optionally surface the mode toggle in the touch settings screen.

Reuse the P4's beat→color selection idea (downbeat = accent, else beat color);
if practical, factor a tiny pure "which color for this beat" helper so the S3 dot
and the P4 strip share one host-tested rule.

## Notes

- Pure UX/parity; no protocol or timing impact.
- Bar position / downbeat detection already exists for the quantized phase
  (LNK-019 `quantum_beats`); reuse it rather than re-deriving.
- Ties the S3 and P4 visual language together — part of the shared device-family
  story ([[p4hub-product-vision]], [[kitchensync-product-name]]).
- Cross-refs: P4 LED config is `led_mode` / `led_fade` / `led_beat_color` /
  `led_accent_color` in `ks_config.h`; the P4 renderer is `metro_strip.c`.
