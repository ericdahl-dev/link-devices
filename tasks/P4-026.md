## Goal

Lock to a **sample-accurate clock** sent from a DAW: a plugin emits an audio-rate
sync signal into a **line input**, decoded here to a jitter-free, drift-free clock
that drives Link + all outputs. The tightest possible DAW sync.

## Inspiration

JMK CLOCKstep:MULTI "Sample Accurate Clocking" — a DAW plugin sends sample-accurate
clock through the audio interface into the box's audio input to resolve USB-MIDI
jitter/latency (https://jmkmusicpedals.com/products/clockstep-multi). Their users
call the audio-in sync a favourite feature.

## Hardware gate

This needs a **line-level input**. Our P4-NANO exposes only the **onboard mic** on
the ES8311 codec ADC (acoustic pickup, P4-020), not a line jack. Options:
- Add a **line-in jack wired to the codec ADC** (small hardware mod), switchable
  with the mic; or
- Accept line level into the existing ADC path if the board's mic front-end
  tolerates it (verify levels / risk of clipping).

So this is **hardware-gated** — file now, build after the mic path (P4-020) exists
and a line input is available.

## Scope

- Define/borrow the sync-signal format (their plugin's; or our own audio-rate
  clock encoding). A DAW-side plugin is out of scope for the firmware repo — start
  by decoding a known tone/pulse train.
- Pure decoder: audio buffer -> recovered clock phase/tempo, sample-locked.
- Feed the tempo-source arbiter (P4-011 s2 / P4-020), highest-priority when locked.

## Deliverable

With a line signal present, the P4's clock is sample-locked to the DAW (measurably
lower jitter than USB-MIDI). Blocked on a line input; sequence after P4-020.
