## Goal

Create the `esp32/X32FaderDisp/` Arduino sketch — an ESP32-S3 firmware that
subscribes to a mixer over OSC and, in a later task (FDR-002), writes each
channel's live fader value **in dB onto its scribble strip**.

> **Model scope:** this is an **X32/M32** feature (port `10023`). The XR18/XAir
> has no physical scribble strips, so the firmware connects but has no visible
> effect there. Default config targets X32. See
> [`docs/x32-osc-protocol.md`](../../docs/x32-osc-protocol.md).

This task is scaffold + handshake only — no dB logic yet.

## Acceptance Criteria

- [ ] `X32FaderDisp/X32FaderDisp.ino` exists. `setup()` connects to WiFi,
      `/xinfo` handshake to `g_config.mixer_ip` on the model port, starts
      `osc_listener`, and registers a callback that just logs every received
      message. `loop()` services the listener (keepalive).
- [ ] `build_opt.h` matches the other S3 sketches (`-DBOARD_HAS_PSRAM` only if
      the chosen board has it; FaderDisp needs no PSRAM).
- [ ] `config.h` contains FDR constants (see below).
- [ ] All symlinks above created and resolve (`ln -s ../X32Link/<file> <file>`).
- [ ] Compiles for the Super Mini / XIAO FQBN used by X32Link; flash % + DRAM %
      recorded in Progress.
- [ ] Serial on boot: `[FDR] connected`, `[FDR] handshake OK`, then a line per
      OSC message once a client moves a fader on the emulator.

## config.h contents

```c
#pragma once
#define FDR_NAME_REFRESH_MS   100   // min ms between scribble writes per channel
#define FDR_CHAN_COUNT        32    // X32 channels (16 for XR18)
#define FDR_RESTORE_ON_EXIT   1     // save/restore original names (FDR-002)
#define FDR_XREMOTE_INTERVAL  9000  // ms between /xremote keepalives
```
