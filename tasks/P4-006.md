## Goal

Click out the P4-NANO's **onboard speaker** on each beat, with an accent on the
bar-1 downbeat, driven by the Link phase pipeline. Self-contained feature that
showcases the P4's audio (the S3 has no audio out) and is genuinely useful as a
standalone reference-click for a room.

## Why

A hardware metronome locked to the Link session — no laptop, no phone — is a
natural P4-only feature and a clean demo of the phase clock driving real output.

## Reuse

Beat/phase already exist as pure logic: `beat_synth` (LNK-033 free-run beat edge
detect) / `clock_ticker` BarReset (bar boundary) / `tempo_source_phase()`. The
metronome scheduler is pure (when to click, accent-vs-plain); only the audio glue
(codec/I2S/PWM tone burst) is platform-specific. Keep the click-decision logic in
a host-tested pure module.

## Open questions

- **Audio path:** onboard speaker — codec chip? I2S DAC? PWM? Which driver/pins
  on the Waveshare P4-NANO (check the schematic). (P4-001 open question.)
- Click sound: short tone burst vs sample; accent = higher pitch / louder.
- Config: on/off + accent toggle via web UI (per ADR-0004, config lives in the
  web UI; touch/real-time controls are separate).

## Depends on

- P4-004 — needs a live tempo/phase (Link). A fixed-BPM test mode can unblock the
  audio-path work before P4-004 lands.

## Deliverable

Speaker clicks on the beat, accents bar 1, locked to the Link session; on/off +
accent configurable. Click-decision logic host-tested.

## Progress

Implemented on `feat/p4-006-metronome` (off `master`; does NOT depend on the
P4-009 phase-lock work).

**Audio path (resolved open question):** the ESP32-P4-NANO onboard speaker is an
**ES8311 codec (I2C addr 0x18) + NS4150B power amp**, driven over I2S. Pins taken
from the Waveshare `esp32-p4-platform` `12_I2SCodec` example:

- I2C: `SCL=GPIO8  SDA=GPIO7` (I2C0)
- Power-amp enable: `GPIO53` (active-high)
- I2S: `MCLK=GPIO13  BCLK=GPIO12  WS=GPIO10  DOUT=GPIO9` (DIN=GPIO11, mic, unused)

None collide with the C6/ESP-Hosted SDIO pins (CLK18/CMD19/D14-17, rst54) or USB
host. Pins are LOGGED at boot for the user to confirm against the board.

**Pure logic (host-tested, red-green TDD):** `X32Link/metronome.{h,c}` — composes
`clock_ticker` (ppqn=1, one pulse/beat) + `BarReset` (bar downbeat = accent),
both reused unchanged. Suite `test/test_metronome.c` (7 tests). Config extended
in `ks_config.{h,c}` (`metronome_enable`/`metronome_accent`) with tests.

**Glue:** `KitchenSync/main/metronome_audio.{c,h}` (ES8311/I2S init + pre-rendered
tone-burst player task); wired into `ks_main.c` (self-contained — advances the
beat clock and clicks even with no USB-MIDI device); toggles added to the web UI.
Also fixed a latent watchdog: `CONFIG_FREERTOS_HZ=1000` so the "1 ms" clock_out
loop's `vTaskDelay(pdMS_TO_TICKS(1))` is a real yield (was 0 ticks at 100 Hz →
IDLE0 starvation).

**Verified:** host tests green; clean `idf.py build` (esp32p4, ES8311 managed
component); flashed to the P4-NANO — clean boot, no panic/WDT, codec inits
("metronome audio ready"), and with a synthetic 120 BPM tempo the click cadence
logs beat-accurate with ACCENT on beats 4/8/12/16/20 (bar downbeats).

**Remaining (needs the user — audio can't be self-verified):** enable Metronome
in the web UI with a live Link session and LISTEN — confirm a click on each beat,
a distinct higher/louder accent on the bar-1 downbeat, and no pops. If silent,
re-check the logged ES8311/I2S pin map against the board (some P4 boards need
DOUT/DIN swapped). NOTE: the accent is periodic every 4 beats from the local
free-run clock (assumes 4/4 — Link carries no meter), not session-phase-aligned;
true downbeat alignment would ride the P4-009 phase pipeline.
