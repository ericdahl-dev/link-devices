## Goal

Send MIDI **transport** messages out the USB-MIDI host alongside the 24-PPQN
clock (P4-005): **Start `0xFA`**, **Stop `0xFC`**, **Continue `0xFB`**, driven by
the Ableton Link session's play/stop state — so downstream gear (arps, sequencers)
runs and stops in lockstep, not just tracks tempo.

## Reuse

The pure `usb_midi_pack.c` already encodes single-byte messages, so
`usb_midi_pack_single(cable, 0xFA/0xFB/0xFC)` is done — this is transport *policy*
(when to emit Start vs Continue vs Stop) + reading Link's start-stop state.

## Scope

- Read Link transport (is-playing / start-stop-sync) from the gossip — confirm
  `link_protocol.c` exposes it, or extend the parser (pure, TDD'd) to surface a
  play/stop flag + start beat.
- Pure transport state machine (TDD'd): Link stopped→playing = Start (or Continue
  if resuming mid-song); playing→stopped = Stop. Emit exactly once per edge.
- Glue: emit the packet via `usb_midi_host_send()`; align Start to the clock so
  gear starts on the beat.

## Deliverable

Pressing play/stop in Ableton starts/stops the attached USB-MIDI gear in sync.
Transport state machine host-tested.
