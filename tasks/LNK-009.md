## Goal

Sort the two firmwares' timing constants into **shared** (truly identical →
one `APP_*` name) vs **input-specific** (differ per source → owned by the
adapter, not a global). This is the constant-side enabler for merging into one
firmware (LNK-010), where there is a single `config.h` but two inputs with
different jitter/poll characteristics.

## Current state

`X32Link/config.h` (`LINK_*`) and `X32MidiClock/config.h` (`MCK_*`) declare the
same knobs under different prefixes:

| Concept            | LINK_ | MCK_ | Identical? | Destination                |
|--------------------|-------|------|-----------|-----------------------------|
| DEFAULT_BPM        | 120.0 | 120.0| yes       | shared `APP_DEFAULT_BPM`     |
| SEND_INTERVAL_MS   | 500   | 500  | yes       | shared `APP_SEND_INTERVAL_MS`|
| REFRESH_BARS       | 1     | 1    | yes       | shared `APP_REFRESH_BARS`    |
| BPM_THRESHOLD      | 0.5   | 1.5  | **no**    | per-input (adapter-owned)   |
| POLL_MS            | 50    | 10   | **no**    | per-input (adapter-owned)   |
| CLOCK_WINDOW       | —     | 24   | MIDI-only | MIDI adapter only            |
| CLOCK_TIMEOUT_MS   | —     | 2000 | MIDI-only | MIDI adapter only            |

## Proposed change

1. **Shared, identical** → one `APP_*` set in the single `config.h`:
   ```c
   #define APP_DEFAULT_BPM       120.0f
   #define APP_SEND_INTERVAL_MS  500
   #define APP_REFRESH_BARS      1
   ```
2. **Per-input** (`BPM_THRESHOLD`, `POLL_MS`) → the input owns them. Expose via
   the TempoSource seam (LNK-008), e.g. `tempo_source_threshold()` and
   `tempo_source_poll_ms()`, so the shared `bpm_task` reads the value for the
   active source. (Link reports 0.5 / 50 ms; MIDI reports 1.5 / 10 ms.)
   `bpm_tracker_init` is then called with the active source's threshold.
3. **MIDI-only** knobs stay behind the MIDI adapter.

## Constraints

- No behaviour change: same values reach the same call sites, just sourced
  per-input instead of via a fixed macro.
- `DEFAULT_WIFI_*` / `DEFAULT_MIXER_IP` / `DEFAULT_MODEL` / `DEFAULT_FX_SLOT`
  already share names — leave them.

## Acceptance Criteria

- [ ] Single `config.h` carries the three shared `APP_*` constants.
- [ ] `BPM_THRESHOLD` and `POLL_MS` are sourced per active input (adapter), not
      a global macro; Link still uses 0.5/50, MIDI still uses 1.5/10.
- [ ] No `LINK_*` / `MCK_*` timing macros referenced in shared code.
- [ ] Firmware compiles (target board per LNK-010).
- [ ] Host tests still pass: `cd test && make`
