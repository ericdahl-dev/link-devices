## Goal

Eliminate `getopt()` from X32.c copy — not available on ESP32. Replace with `x32_apply_defaults()` which sets port, verbose, debug to known-good values.

## Acceptance Criteria

- [ ] `getopt()` loop in `main()` (lines ~878-920 in original) wrapped in `#ifndef ARDUINO` or removed from x32_init()
- [ ] `x32_init()` calls `x32_apply_defaults()` instead
- [ ] `#include <unistd.h>` (provides getopt on Linux) wrapped in `#ifndef ARDUINO`
- [ ] Native tests still pass
- [ ] Sketch compiles with no getopt/optarg errors

## Notes

**File to edit**: esp32/X32_emulator/X32.c (copy)

**getopt block to wrap** (original lines 878-920):
```c
while ((input_intch = getopt(argc, argv, "i:d:v:x:b:f:r:m:h")) != -1) {
    switch ((char)input_intch) {
    case 'i': strcpy(Xip_str, optarg); noIP = 0; break;
    case 'd': sscanf(optarg, "%d", &Xdebug); break;
    case 'v': sscanf(optarg, "%d", &Xverbose); break;
    ...
    }
}
```

**x32_apply_defaults()** already in x32_port.c:
- `Xport_str = "10023"`
- `Xverbose = 1`
- `Xdebug = 0`

**x32_init() signature** (Arduino):
```c
void x32_init(const char *wifi_ip) {
    x32_apply_defaults();
    x32_get_ip(wifi_ip);
    // then socket setup...
}
```

WiFi credentials (SSID/password) go in X32_emulator.ino as `#define` constants — not in X32.c.
