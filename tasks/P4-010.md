## Goal

Make the P4 a Multiclock-class device (ref: https://www.e-rm.de/multiclock/):
**multiple independent clock outputs**, each with its own settings, configured via
the web UI (P4-007):

- **Clock division / multiplication** per output (e.g. /4, ×2). `clock_ticker`
  already emits at a configurable PPQN, so this is largely there.
- **Per-output phase offset / nudge** — the Multiclock's signature feature: shift
  an output early/late in fine increments to latency-compensate analog gear.
- **Shuffle / swing** per output.
- Route each output to a **USB-MIDI cable** (0–3, e.g. Midihub USB A–D) and later
  DIN/analog.

## Reuse / architecture

Pure, host-tested logic layered on `clock_ticker` + `beat_clock`:
- Extend the tick scheduler with a per-output phase term + swing (new pure funcs,
  TDD'd); keep the base engine.
- A per-output config struct in the P4Hub NVS config; edited in the web UI.

## Deliverable

N configurable clock outputs (division, phase-nudge, swing, cable), set in the web
UI and persisted, each emitting its own USB-MIDI clock. Scheduler logic host-tested.

## Notes

- Depends on the web UI (P4-007) for the config surface and on P4-005 for the
  base clock path. Phase-nudge pairs well with P4-009 (real phase reference).
