## Goal

Give P4Hub a web UI that matches the shipped **X32Link rack-panel aesthetic**
(dark rack unit, screws, green-LED accent `#b6ff36`, DSEG 7-seg tempo readout,
segmented controls / toggles / "Write & Reboot" button; see
`X32Link/web_config.cpp`). It's the P4's config surface per ADR-0004 (web UI =
full config; touch/real-time is separate) and the home for the Multiclock-style
per-output settings (P4-010).

## Scope (v1)

- **esp_http_server** (ESP-IDF) glue — the P4 is ESP-IDF, so not the Arduino
  `WebServer` X32Link uses. Serve `/` (page), `/status` (JSON), `/save` (POST).
- **Live status**, styled like X32Link: 7-seg **Link BPM**, beat dot, peers,
  USB-MIDI device state, clock running/TX. Poll `/status` at 1 Hz (reuse the
  X32Link JS shape).
- **Runtime config → NVS** (new for P4Hub): move WiFi SSID/pass off compile-time
  Kconfig to web-set + NVS; clock-out enable + USB cable select. Save + reboot.
- **Pure, host-tested builders** per ADR-0003: page/status render + form parse in
  a `.c` with a Unity suite; `esp_http_server` is thin glue.

## Reuse

Lift the X32Link HTML/CSS template + the `/status` JS (phase-locked beat dot) and
the `web_status_json.c` pattern. Fonts load from CDN when online (same as
X32Link), mono fallback on the first-boot SoftAP.

## Deliverable

P4Hub serves a rack-panel page on its IP showing live Link tempo + device state,
with WiFi + clock config saved to NVS. Verified in a browser on the network.

## Notes

- Foundation for P4-010 (Multiclock-style per-output clock config).
- Consider a first-boot SoftAP + captive portal like X32Link so WiFi can be set
  with no serial (follow-up if not in v1).
