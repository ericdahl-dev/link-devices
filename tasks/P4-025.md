## Goal

Smarter transport + a global timing offset:
- **Quantize**: start on the next bar line, not immediately.
- **Resync**: re-align downstream gear to the session downbeat on demand.
- **Independent clock/transport**: option to keep clock running regardless of
  Start/Stop.
- **Latency offset**: a global +/- millisecond nudge to null latency between the
  P4 and downstream sequencers / DAW.

## Inspiration

JMK CLOCKstep:MULTI "Transport Control" (Start/Stop/Pause, Quantize, Resync,
independent clock/transport) and "Latency Compensation/Offset" (+/- ms)
(https://jmkmusicpedals.com/products/clockstep-multi).

## Current state

`transport` sends MIDI Start/Stop on the Link play-state edge (P4-008). No
quantize/resync/independent modes. Per-output phase nudge exists (milli-beats,
P4-010) but there's no single global ms offset.

## Scope

- Extend the pure `transport` module with a mode (immediate / quantize-to-bar /
  independent) using the bar position (needs P4-022 meter). Host-tested.
- Global `latency_offset_ms` config applied to the emitted clock/transport timing.
- Web UI: transport-mode select + latency-offset field.

## Deliverable

Start quantized to the bar; a global ms offset shifts all outputs. Host-tested
mode logic.
