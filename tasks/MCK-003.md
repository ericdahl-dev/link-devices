## Goal

Convert the raw stream of 0xF8 MIDI Clock pulses (24 per quarter note) into a
stable BPM value. Average over one full beat (24 pulses) to smooth jitter.
Detect clock stop (timeout) and stop sending OSC when the DAW pauses.

## Acceptance Criteria

- [ ] `midi_bpm_init()` resets state
- [ ] `midi_bpm_update()` reads the pulse ring buffer and returns updated BPM,
      or 0.0 if fewer than 24 pulses seen yet or clock timed out
- [ ] BPM accurate to ±0.5 BPM across range 40–250 BPM (verified with tone generator)
- [ ] Timeout: if no pulse received for `MCK_CLOCK_TIMEOUT_MS`, BPM resets to 0.0
      and OSC sending stops
- [ ] Handles tempo changes — new BPM stable within 1 beat of change
- [ ] Reuses `bpm_tracker_update()` from shared module for change-threshold gating
- [ ] Files: `midi_bpm.h`, `midi_bpm.cpp`

## Notes

**MIDI Clock math** (24 pulses per quarter note = 24 PPQN):
```
period_us  = average interval between last 24 pulses
beat_us    = period_us × 24
BPM        = 60,000,000 / beat_us
```

**Implementation** — sliding window over the timestamp ring from MCK-002:
```cpp
float midi_bpm_update() {
    if (clock_pulse_count < 24) return 0.0f;

    // Take timestamps of the last 24 pulses
    uint32_t newest = clock_ring[(clock_head - 1) % RING_SIZE];
    uint32_t oldest = clock_ring[(clock_head - 24) % RING_SIZE];
    uint32_t beat_us = newest - oldest;   // wraps correctly for uint32_t

    if (beat_us == 0) return 0.0f;
    return 60000000.0f / (float)beat_us;
}
```

**Timeout detection** — check wall time against last pulse timestamp:
```cpp
if (micros() - clock_ring[(clock_head - 1) % RING_SIZE] > MCK_CLOCK_TIMEOUT_MS * 1000UL)
    return 0.0f;
```

**Why 24-pulse window**: averaging over exactly one beat mirrors the musical
tempo; shorter windows are jittery, longer windows lag behind tempo changes.
With `MCK_BPM_THRESHOLD = 0.5`, OSC only fires when BPM drifts more than ±0.5
from the last sent value (gated by `bpm_tracker_update()`).

**Jitter source**: USB MIDI timing jitter on most hosts is < 1 ms, giving
< 0.1 BPM error at 120 BPM. Software DAWs vary — Ableton's clock is reliable;
some DAWs can have 2–5 ms jitter which at extreme tempos may show as ±1 BPM.
The 24-pulse average handles this well.
