## Goal

Confirm the X32 emulator running on XIAO ESP32-S3 responds correctly to OSC messages from a real client over WiFi UDP port 10023.

## Acceptance Criteria

- [ ] Board joins WiFi and prints assigned IP on serial
- [ ] `/info` OSC query from Linux client returns valid X32 info response
- [ ] `/status` query returns correct status blob
- [ ] `/xremote` registers a remote client; subsequent param changes are broadcast
- [ ] `/shutdown` OSC command saves state to LittleFS; board restart reloads state
- [ ] X32-Edit (or X32_Command CLI) can connect and query parameters without errors

## Notes

**Test client**: src/build/X32_Command (already compiled Linux binary)
```
./X32_Command -i <ESP32_IP>
```

**Manual OSC test** (Python):
```python
import socket, struct

def osc_str(s):
    b = s.encode() + b'\x00'
    return b + b'\x00' * ((4 - len(b) % 4) % 4)

def send_osc(ip, port, path, typetag=b',', args=b''):
    pkt = osc_str(path) + osc_str(typetag.decode()) + args
    s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
    s.sendto(pkt, (ip, port))
    s.settimeout(2)
    return s.recvfrom(1024)

resp = send_osc('192.168.x.x', 10023, '/info')
print(resp)
```

**WiFi credentials**: set WIFI_SSID / WIFI_PASS in X32_emulator.ino.

**Known risks**:
- PSRAM latency: large X32CfgMain data in PSRAM may cause slower lookup — profile if /node queries are slow
- UDP buffer: ESP32 lwIP default UDP buffer 1460 bytes; X32 responses can be larger. May need `lwip_setsockopt(fd, SOL_SOCKET, SO_RCVBUF, ...)`.
- `select()` timeout 10ms (from original) — fine for FreeRTOS; yields to WiFi stack between calls.
