## Goal

Make the ESP32-S3 enumerate as a class-compliant USB MIDI device so any DAW
(Ableton, Logic, etc.) can select it as a MIDI output and send clock to it.
Capture incoming 0xF8 (MIDI Clock) messages and expose them to the BPM
calculation task via a shared timestamp ring buffer.

## Acceptance Criteria

- [ ] Device enumerates on macOS/Windows/Linux with no driver install
- [ ] Device name visible in Ableton MIDI prefs as "X32MidiClock" (or similar)
- [ ] `midi_clock_init()` sets up TinyUSB MIDI and installs the 0xF8 callback
- [ ] Callback stores `micros()` in a ring buffer (capacity = `MCK_CLOCK_WINDOW * 2`)
- [ ] `midi_clock_pulse_count()` returns total pulses received (for BPM task to poll)
- [ ] No clock → device still enumerates and sits idle (no crash, no WDT reset)
- [ ] Files: `midi_clock.h`, `midi_clock.cpp`

## Notes

**Arduino ESP32 USB MIDI API** (ESP32 Arduino core ≥ 2.0.14):
```cpp
#include <USB.h>
#include <USBMIDI.h>

USBMIDI MidiUSB;

void setup() {
    MidiUSB.begin();
    USB.productName("X32MidiClock");
    USB.manufacturerName("X32Link");
    USB.begin();
}
```
`USBMIDI` uses TinyUSB under the hood — no separate TinyUSB library needed when
using the ESP32 Arduino core.

**Reading MIDI clock** — poll in a FreeRTOS task at high frequency or use the
`onEvent` callback:
```cpp
MidiUSB.onEvent([](uint8_t cable, uint8_t byte0, uint8_t byte1, uint8_t byte2) {
    if (byte0 == 0xF8) {          // MIDI Clock pulse
        clock_ring[clock_head++ % RING_SIZE] = micros();
        clock_pulse_count++;
    }
});
```
USBMIDI callbacks run in a USB task context — ring buffer access must be
interrupt-safe (use `portENTER_CRITICAL` or make the ring index `volatile`).

**USB vs UART debug**: With USB-OTG active, `Serial` is the USB CDC port — it
won't appear until the host connects. Use `Serial0` (UART0, pin 43/44 on S3)
for reliable boot debug. Or print nothing and use the onboard LED as a heartbeat.

**Device naming**: Set `USB.productName()` before `USB.begin()`. Ableton reads
the USB product string to populate the MIDI device list.
