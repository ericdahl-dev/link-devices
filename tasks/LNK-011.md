## Goal

Turn the "should I send OSC now, and with what value?" decision — currently
loose, duplicated, and untested inside `bpm_task` — into one pure, host-tested
module. This is the exact logic that was debugged by hand on hardware
(the bpm=0 gate, the per-bar refresh, the change threshold); it deserves a test
surface like `link_protocol` already has.

## Current state

The decision lives inline in `bpm_task` (both `.ino` files today):

```c
bool changed = bpm > 0.0f && bpm_tracker_update(bpm);
bool refresh = bpm > 0.0f && active &&
               now - last_bar_ms >= (uint32_t)(4*REFRESH_BARS*60000.0f/bpm);
if ((changed || refresh) && (now - last_send_ms) >= SEND_INTERVAL_MS) {
    osc_send_bpm(bpm); last_send_ms = now; ...
}
```

It tangles: change detection (`bpm_tracker`), per-bar refresh timing, the
minimum send interval, and the bpm=0 "stop sending, don't push 0 to mixer"
rule. No locality (duplicated), no test surface.

## Proposed change

A pure C module `bpm_publisher.{c,h}` — small interface, all the timing nuance
behind it. Holds its own `last_send_ms` / `last_bar_ms` / tracker state.

```c
typedef struct { bool send; float bpm; } PublishDecision;

void  bpm_publisher_init(float threshold, uint32_t send_interval_ms,
                         int refresh_bars);
// Feed the latest reading each tick; returns whether to emit OSC now.
PublishDecision bpm_publisher_step(float bpm, bool active, uint32_t now_ms);
```

`bpm_task` becomes: read the seam, call `bpm_publisher_step`, and if
`.send` call `osc_send_bpm(.bpm)`. The `threshold` comes from the active input
(LNK-009), so per-source jitter handling lives here cleanly.

## Constraints

- Pure C, no Arduino deps — buildable and testable on host (follow the
  existing `test/test_link_protocol.c` + weak-`millis` pattern).
- Preserve current behaviour exactly: bpm=0 ⇒ no send (mixer holds last value);
  send only on change-past-threshold OR per-bar refresh; respect min interval.

## Acceptance Criteria

- [ ] `bpm_publisher.{c,h}` exists; pure C; no Arduino include.
- [ ] `bpm_task` uses it; no send-policy arithmetic remains inline.
- [ ] New `test/test_bpm_publisher.c` added to `test/Makefile`, covering:
      change-triggers-send, jitter-below-threshold-suppressed,
      per-bar-refresh-send, min-interval-throttle, and **bpm=0 ⇒ no send**.
- [ ] `cd test && make` builds and passes all suites.
- [ ] Firmware compiles for the target board and behaves identically on-device.
