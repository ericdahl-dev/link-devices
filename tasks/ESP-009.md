## Goal

When KitchenSync is healthy, joined to a Link session, and merely waiting for
transport to start, it is completely silent and dark -- indistinguishable from
a dead board. Give it a heartbeat so "stopped" reads as "ready", not "broken".

## How this was found

2026-07-10, on the bench. Enabling Start/Stop sync in Ableton made the
metronome and LED strip stop, which looked like a hardware fault. It was not:
`ks_tick.c:56` gates the click (and therefore the visual metronome) on
`in->tp_playing`. With Start/Stop sync OFF, Live publishes no transport, the
device free-runs, and everything blinks. Turn it ON with the transport
stopped, and the gate correctly closes. Correct behavior, alarming
presentation -- it cost a real debugging session to rediscover.

## Scope

- While `have_session && !tp_playing`: a slow, dim LED pulse (~0.5Hz) on the
  WS2812 strip. Distinct from the beat chase, obviously "idle" not "playing".
- Silence stays silence: no audible standby tick. The speaker should not
  chirp at an idle rig.
- Nothing when there is no session at all -- that state already has the
  WiFi-down triple-blink convention on the S3 (LNK-023) and should stay
  visually distinct from "connected, waiting".

## Design notes

`metro_strip.c` is pure and host-tested (11 tests) -- the standby pattern
belongs there as another mode alongside chase/flash/fill, chosen by the
caller, not as a special case inside `ks_led.c`. `ks_tick_step` already knows
`tp_playing` and returns a plan; the plan can carry a `standby` flag the same
way it carries `click`/`click_accent`. That keeps the decision pure and
testable and leaves `ks_led.c` as dumb glue (ADR-0003).

## Acceptance Criteria

- [ ] Session joined + transport stopped -> slow dim pulse on the strip
- [ ] Transport playing -> normal beat chase, no trace of standby
- [ ] No session -> neither pattern (stays distinct from standby)
- [ ] Metronome stays silent in standby
- [ ] Pure standby pattern host-tested in `test/test_metro_strip.c`
- [ ] Verified on hardware with Ableton Start/Stop sync toggled both ways

## Notes

- Consider the same idea for the S3 boards' single beat LED later; this
  ticket is P4/strip only.
- Cheap and genuinely field-relevant: this is the failure mode a customer
  would report as "your box is dead".
