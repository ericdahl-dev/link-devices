## Why

"What identity does this device advertise" is scattered as inline literals across three glue
files, and half of it has no owner and no test.

- **Hostname derivation** is duplicated three times. `ks_hostname` (`KitchenSync/main/ks_hostname.c`)
  is pure and host-tested — but its prefix is hardcoded `"kitchensync-"` and **only the P4 uses
  it**. The two Arduino firmwares re-derive the same `<prefix>-%02x%02x` shape inline:
  - `X32Link/X32Link.ino` — `snprintf(g_ks_host, ..., "x32link-%02x%02x", mac[4], mac[5])`
  - `KitchenSyncTouch/KitchenSyncTouch.ino` — `"kstouch-%02x%02x"`
- **The mDNS TXT tuple `{dev, model, target, fw}`** (added in ESP-031) is typed out **three
  times** — `X32Link.ino`, `KitchenSyncTouch.ino`, `wifi_link.c` — with `dev="kitchensync"` and
  `target=CONFIG_IDF_TARGET` identical at every site, only `model` varying. It has **zero test
  coverage** anywhere.

`ESP-014` already names the first half of the fix: *"Generalise `ks_hostname` to take a prefix
… rather than copying."* This ticket does that and folds in the TXT tuple.

This is the canonical shallow-duplication case: adding one TXT key (as ESP-031 did) means editing
three files by hand; the next key will too.

## What to build

A pure, host-tested identity module — the sanctioned home for shared pure C is the `X32Link/`
sketch root (ADR-0006/0007), symlinked into `KitchenSyncTouch/`, compiled-by-path into
`KitchenSync/`:

```c
// device_identity.h
typedef struct { const char* dev; const char* model; const char* target; const char* fw; } DeviceIdentity;
void device_hostname(const char* prefix, const uint8_t mac[6], char* out, size_t cap);  // generalises ks_hostname
// (advertise the TXT tuple via one helper so the key set lives in ONE place)
```

Each glue site supplies its `model` (`x32link` / `touch` / `p4`) and prefix; the module owns the
hostname shape, the TXT key names, and their order. Retire `ks_hostname` into this, or keep it as
a thin `prefix="kitchensync-"` wrapper.

## Acceptance criteria

- [ ] `device_hostname(prefix, mac, out, cap)` pure, host-tested (port `test_ks_hostname.c` cases,
      add a per-prefix case). NUL-safe truncation preserved.
- [ ] The `{dev, model, target, fw}` key set is declared in ONE place and host-tested.
- [ ] All three firmwares derive hostname + TXT from the module — no inline `snprintf` hostname,
      no per-file `addServiceTxt`/`mdns_txt_item_t` literal tuple.
- [ ] `dns-sd -L <instance> _http._tcp local` shows the tuple on the two S3 units (hardware).
- [ ] CI green for esp32 / esp32p4 / esp32s3.
