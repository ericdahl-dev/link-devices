## Problem

Held in the hand, the X32Link (Waveshare S3 Touch-LCD-1.47) currently orients so
the USB-C cable exits the **top** — the cable runs up and over the hand, which is
awkward and strains the connector. The natural grip has the cable exiting the
**bottom**, running down out of the hand. The display (and touch) need to flip
180° so "up" on screen matches "up" in that grip.

## Fix

Flip the panel 180° and update the touch transform to match — they are coupled
(LNK-014). If only the display rotates, taps land in the wrong place.

- `X32Link/touch_display.cpp:390` — `s_lcd.setRotation(4)` → the 180°-rotated
  variant. LovyanGFX rotations 4–7 are the mirrored set at 0/90/180/270°, so
  180° from `4` is `setRotation(6)`. Verify on hardware (confirm text reads
  upright and the phase wheel sits correctly in the 172×320 portrait frame).
- `X32Link/touch_display.cpp:~417` — the tap mapping is currently the rotation-4
  transform `x = raw.x; y = 319 - raw.y`. A 180° flip inverts the other axis:
  `x = 171 - raw.x` (panel width 172) and `y = raw.y`. Confirm empirically —
  tap each target (STATUS "SET", settings rows, keyboard) and check hits.
- Re-check the phase-wheel / tap-target geometry comments at
  `touch_display.cpp:26`, `:76`, `:88` and the host test `test/test_touch_ui.c`
  if any rect coordinates are hard-coded to the old orientation.

## Notes

- Pure ergonomics; no protocol/timing impact. Low priority.
- Keep it a single constant/transform change so it's easy to flip back if the
  preferred grip changes. Consider whether this should eventually be a
  user-settable orientation (config field) rather than a compile-time rotation.
- Touch pinmap / rotation background: see [[waveshare-s3-touch-lcd-147-board]]
  (the GPIO38=LCD_SCL trap) and LNK-014.
