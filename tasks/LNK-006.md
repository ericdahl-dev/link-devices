## Goal

Confirm the complete chain — DAW tempo change → Link session → ESP32 → XR18
FX delay time — works correctly across the expected BPM range and survives
common failure conditions (WiFi drop, mixer power cycle, DAW disconnect).

## Acceptance Criteria

- [ ] DAW tempo change is reflected in XR18 FX delay time within 500 ms
- [ ] BPM range 60–200 BPM tested; delay time matches `bpm_to_normalized()` formula
- [ ] Two Link peers (DAW + one other app) both influence session tempo correctly
- [ ] WiFi drop + reconnect: firmware recovers and resumes sending OSC without reboot
- [ ] XR18 power cycle: firmware detects UDP failures gracefully, retries on next BPM change
- [ ] 30-minute soak test: no crashes, no stuck BPM, no memory leak (Serial shows free heap stable)

## Notes

**Firmware scope**: `esp32/X32Link/` only. Do NOT touch `esp32/X32_emulator/`.

**Test setup**:
1. XR18, ESP32, and DAW all on same 2.4 GHz WiFi subnet
2. Load a delay effect (e.g. `DELAY` or `MONO DLY`) into FX slot `LINK_FX_SLOT`
3. Open X-Air Edit and watch the delay time parameter for slot in real time
4. Enable Link in DAW, confirm ESP32 Serial shows `Link peers: 1`
5. Change DAW tempo; watch X-Air Edit delay time update

**Manual OSC verification** (bypass firmware, confirm XR18 path works):
```bash
# From Linux — sends /fx/1/par/02 ,f 0.1667 (120 BPM equivalent)
python3 -c "
import socket, struct
path = b'/fx/1/par/02\x00\x00\x00\x00'  # 16 bytes
tag  = b',f\x00\x00'
val  = struct.pack('>f', 0.1667)
s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
s.sendto(path + tag + val, ('192.168.x.x', 10024))
"
```

**Soak test** — add to `loop()`:
```cpp
Serial.printf("Heap: %d  BPM: %.2f  Peers: %d\n",
              ESP.getFreeHeap(), g_current_bpm, link_peer_count());
```
Run for 30 min; heap should not decrease monotonically.

**Known risks**:
- UDP `endPacket()` returns 0 on failure (XR18 unreachable) — log but do not
  block; next BPM change will retry
- Link session tempo can drift ±0.001 BPM due to floating-point — the
  `LINK_BPM_THRESHOLD` of 0.5 BPM suppresses spurious OSC writes
- If delay slot is wrong type (e.g. reverb instead of delay), `/fx/{slot}/par/02`
  changes a different parameter — verify FX type in X-Air Edit first
