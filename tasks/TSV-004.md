## Goal

Implement `esp32/X32ToastSaver/teq_notch.cpp` and `teq_notch.h`: receive a `FeedbackEvent` from the FFT detector, apply a TEQ notch cut to the XR18 via OSC, track the 31 GEQ bands, and release notches after sustained quiet.

## Public interface (teq_notch.h)

```cpp
#pragma once
#include "fft_detector.h"
#include <stdbool.h>

/* Initialize notch state. Call from setup() after WiFi and handshake. */
void teq_notch_begin(void);

/* Process one FFT frame:
 *   - If evt is non-null, apply a new notch for that GEQ par (unless already notched).
 *   - Check all active notches for release eligibility.
 *   - Called every loop() iteration; pass evt=nullptr when no feedback detected. */
void teq_notch_update(const FeedbackEvent *evt);

/* Restore all active notches to flat (0.5). Call on shutdown / config change. */
void teq_notch_restore_all(void);

/* Return active notch count (0–31). For display. */
int teq_notch_active_count(void);

/* Fill par_active[31] with 1 for each notched band, 0 otherwise. For display. */
void teq_notch_get_state(uint8_t par_active[31]);
```

## OSC TEQ path

TEQ in FX slot (from `g_config.fx_slot`, default 4):
```
/fx/{slot}/par/{nn} ,f {float_val}
```
- `{nn}` = zero-padded 2-digit par number (01–31)
- flat = 0.5, notch = `(cut_dB + 15.0) / 30.0`  →  −6 dB = 0.3

Use the existing `osc_out.c` building functions and send via WiFiUDP to `g_config.mixer_ip:10024`.

Add a new function to `osc_out.c` (or a local function in `teq_notch.cpp`):

```c
/* Build /fx/{slot}/par/{nn} ,f {val} — returns packet length */
int osc_build_teq_band(uint8_t *buf, int slot, int par, float val);
```

Packet structure:
```
path = "/fx/N/par/NN\0\0\0\0" (padded to 4-byte boundary)
tag  = ",f\0\0"               (4 bytes)
val  = float32 big-endian      (4 bytes)
```

## Notch state

```cpp
typedef struct {
    bool   active;
    float  cut_val;          /* OSC float value sent */
    unsigned long notched_ms; /* millis() when notched */
    int    release_hold;     /* consecutive frames below release threshold */
    float  peak_excess_db;   /* excess dB at time of notch (for logging) */
} BandState;

static BandState s_bands[31];  /* index 0 = GEQ par 01 */
```

## Release logic

Release threshold: `TSV_THRESHOLD_DB / 2.0f` dB below the notched band's noise floor level. Since `teq_notch` doesn't have direct access to the power spectrum, `FeedbackEvent` includes `excess_db`. Track whether a new event fires for the same `geq_par` each frame — if no event fires for that par for `TSV_RELEASE_SEC * 20` consecutive frames (approximately), release it.

Simpler and correct: release a band if no `FeedbackEvent` targets it for `(int)(TSV_RELEASE_SEC * 20)` consecutive frames. This is the same approach used in `XAir_ToastSaver.c`.

```cpp
void teq_notch_update(const FeedbackEvent *evt) {
    /* Notch a new band */
    if (evt) {
        int j = evt->geq_par - 1;
        if (!s_bands[j].active) {
            float cut_val = (TSV_CUT_DB + 15.0f) / 30.0f;
            send_teq_osc(g_config.fx_slot, evt->geq_par, cut_val);
            s_bands[j].active        = true;
            s_bands[j].cut_val       = cut_val;
            s_bands[j].notched_ms    = millis();
            s_bands[j].release_hold  = 0;
            s_bands[j].peak_excess_db = evt->excess_db;
            Serial.printf("[TSV] notch par=%02d (%.0fHz) cut=%.0fdB excess=%.1fdB\n",
                evt->geq_par, evt->freq_hz, TSV_CUT_DB, evt->excess_db);
        }
    }

    /* Release check: any band not targeted this frame gets a hold count increment */
    int release_frames = (int)(TSV_RELEASE_SEC * 20.0f);
    for (int j = 0; j < 31; j++) {
        if (!s_bands[j].active) continue;
        bool targeted = (evt && (evt->geq_par - 1) == j);
        if (!targeted) {
            s_bands[j].release_hold++;
            if (s_bands[j].release_hold >= release_frames) {
                send_teq_osc(g_config.fx_slot, j + 1, 0.5f);
                float elapsed = (millis() - s_bands[j].notched_ms) / 1000.0f;
                Serial.printf("[TSV] release par=%02d after %.1fs\n", j + 1, elapsed);
                s_bands[j].active = false;
                s_bands[j].release_hold = 0;
            }
        } else {
            s_bands[j].release_hold = 0;  /* still triggering — reset hold */
        }
    }
}
```

## Acceptance Criteria

- [ ] `teq_notch.h` defines the five-function interface above.
- [ ] `teq_notch.cpp` implements all five. `osc_build_teq_band()` builds the correct OSC packet.
- [ ] `teq_notch_begin()` called from `setup()` after WiFi handshake.
- [ ] `loop()` passes FFT detector output directly to `teq_notch_update()`.
- [ ] On a feedback event: Serial prints `[TSV] notch par=NN (XXXHz) cut=-6dB excess=YY.YdB`.
- [ ] After `TSV_RELEASE_SEC` seconds without re-triggering: Serial prints `[TSV] release par=NN after Z.Zs`.
- [ ] On SIGINT / power-off (not easily caught on ESP32 — instead add a button or call `teq_notch_restore_all()` from a long-press handler in a later task): all active bands restored to 0.5.
- [ ] Confirmed with XR18: `/fx/4/par/18` changes when a 1 kHz tone triggers detection.

## Notes

- `teq_notch_get_state()` and `teq_notch_active_count()` are for the display task (TSV-006); implement them now but they'll be unused until then.
- The `send_teq_osc()` helper (internal static function) wraps `osc_build_teq_band()` + WiFiUDP send. Keep it simple — `WiFiUDP.beginPacket / write / endPacket`.
- Do not reuse `osc_sender.cpp` (which is BPM-specific from X32Link). Add `osc_build_teq_band()` as a new function in a local `teq_notch.cpp` helper rather than modifying the shared `osc_out.c`.
