## Goal

Wire WiFi startup and Link session init into the sketch main loop. Verify that
the ESP32 appears as a Link peer to other devices on the same network.

## Acceptance Criteria

- [ ] `setup()` connects WiFi then calls `link_init(LINK_DEFAULT_BPM)` + `link_enable(1)`
- [ ] Serial prints peer count every 5 s: `Link peers: N`
- [ ] Peer count increments when a Link-enabled app (iOS, DAW) joins the same WiFi network
- [ ] Peer count decrements when that app leaves
- [ ] WiFi reconnect loop: if WiFi drops, sketch re-joins without reboot

## Notes

**Firmware scope**: `esp32/X32Link/` only. Do NOT touch `esp32/X32_emulator/`.

**Init order matters**: Link must be started AFTER WiFi is connected so it
binds to the correct interface IP. Call `link_enable(1)` only after
`WiFi.status() == WL_CONNECTED`.

**Link multicast**: Link peer discovery uses IPv4 multicast group
`224.76.78.75:20808`. The ESP32 WiFi stack joins this group automatically
when the Link SDK calls `ip_add_membership` via POSIX sockets — no manual
multicast setup needed if the SDK compiles correctly.

**Validation method** — simplest test on the same WiFi:
- Open Ableton Live or any Link-enabled iOS app (e.g. Reason Compact, AUM)
- Enable Link in that app
- Serial on ESP32 should show `Link peers: 1` (or 2 if counting self)

**WiFi reconnect pattern**:
```cpp
void loop() {
    if (WiFi.status() != WL_CONNECTED) {
        link_enable(0);
        WiFi.reconnect();
        while (WiFi.status() != WL_CONNECTED) delay(500);
        link_enable(1);
    }
    // ... rest of loop
}
```

**2.4 GHz only**: the XIAO ESP32-S3 does not support 5 GHz. Ensure `WIFI_SSID`
in config.h refers to the 2.4 GHz band. The XR18 and the DAW must also be on
the same subnet for OSC to reach the mixer.

## Progress

**Strategy pivot** (LNK-002 context exhaustion): The Link SDK ASIO port hit
four unsolvable linker errors under arduino-cli (`pipe`, `pause`,
`__atomic_is_lock_free`, `_ZTINSt3_V214error_categoryE`). Root cause: vanilla
chriskohlhoff/asio has no ESP32 networking backend, and precompiled arduino-esp32
libstdc++ lacks RTTI typeinfo for `std::error_category`. Pivoted to parsing
the Link gossip protocol directly — passive listener is sufficient for the
BPM-read-only use case.

**Implementation** (`link_listener.cpp` + `link_listener.h`):
- Joins multicast 224.76.78.75:20808 via `WiFiUdp::beginMulticast`
- Parses `_asdp_v\x01` discovery packets for `Timeline` TLV (key `0x746d6c6e`)
- Extracts `microsPerBeat` (int64 BE) → BPM = 60e6 / microsPerBeat
- Tracks peers by NodeId with 15 s TTL; handles ByeBye eviction
- All SDK-based files removed (`link_bridge.{h,cpp}`, `abl_link_impl.cpp`,
  `posix_stubs.cpp`, `platform_patches/`, `build_opt.h` cleared)

**Build result**: 67% flash / 14% DRAM (clean compile, no linker errors)
