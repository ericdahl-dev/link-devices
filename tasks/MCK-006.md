> **Declined 2026-07-08:** superseded. X32MidiClock retired (LNK-024); X32Link
> owns MIDI-clock mode, hardware-validated in LNK-020.

## Goal

Validate the complete USB-MIDI-clock-to-X32 chain on real hardware: ESP32-S3
plugged into a MacBook running Ableton, connected over WiFi to an XR18.
Confirm tempo accuracy, tempo change response, and graceful handling of
DAW start/stop.

## Acceptance Criteria

- [ ] Device appears in Ableton MIDI preferences → MIDI Output list as "X32MidiClock"
      with no driver install on macOS
- [ ] Enabling "Clock" output in Ableton and pressing Play causes FX delay time
      on XR18 to update within 1–2 beats
- [ ] BPM accuracy: set Ableton to 80, 120, 140, 180 BPM → X-Air Edit delay time
      matches expected normalised values (see LNK-005 reference table)
- [ ] Tempo change: drag Ableton BPM from 120 → 140 → X32 follows within 1 beat
- [ ] DAW stop: FX delay time stays at last known value (does not reset)
- [ ] DAW start after stop: tempo resumes correctly within 1 beat
- [ ] USB hot-unplug/replug: device re-enumerates without reflash; clock resumes
- [ ] WiFi AP fallback: with no saved credentials, device broadcasts
      "X32MidiClock-Config"; config page saves and restarts correctly
- [ ] Factory reset (BOOT hold 5 s): clears NVS, reboots to AP mode

## Notes

**Test rig**:
- MacBook (or any laptop) running Ableton Live (any version with MIDI clock out)
- ESP32-S3 DevKit flashed with X32MidiClock firmware, USB-C to laptop
- XR18 on same WiFi network as the ESP32's saved SSID
- X-Air Edit open on same machine or phone to observe FX delay parameter

**Ableton MIDI Clock setup**:
Preferences → Link/Tempo/MIDI → MIDI tab → find "X32MidiClock" under Output →
enable "Clock". Press Play in Ableton. Clock starts immediately on Play.

**Checking delay time**: In X-Air Edit, navigate to FX Rack → configured slot →
delay parameter. The value should move as Ableton's BPM changes.

**Alternative test (no XR18 on hand)**: Run `src/X32_emulator` (existing
Python emulator in parent repo) and point the firmware at its IP. It logs
received OSC to stdout — confirm `/fx/{slot}/par/02` arrives with correct values.

**Windows note**: USB MIDI class is natively supported on Windows 10+
(no driver needed). On Windows 7/8, a driver may be required — not a target
platform for this use case.
