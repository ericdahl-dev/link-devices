**Done (PR #27):** `POST /live` applies the live-safe fields (per-output
enable/cable/ppqn/phase/swing, clock-out, metro accent) in place via
`ks_form_apply`, bumps a config-generation counter, and `clock_out_task`
re-primes on change ‚Äî real-time timing, no reboot. Nudge/swing controls got +/-
stepper buttons. wifi + metronome volume/voice stay reboot-only (`/save`).


## Problem

Every config change reboots: `save_handler` writes NVS then unconditionally calls
`esp_restart()`. But `clock_out_task` already reads `g_cfg` live each 1 ms tick ‚Äî
per-output `ppqn` / `cable` / `phase_mbeats` / enables and `metronome_accent` are
applied on the next tick. So phase-nudge (and future swing, P4-013) *could* be
real-time; the reboot is the only thing in the way. That makes dialing in timing
by ear impossible ‚Äî every nudge is a 5-second reboot.

## Solution

A separate `POST /live` endpoint that applies only the **live-safe** fields into
`g_cfg` in place ‚Äî no NVS write, no reboot ‚Äî so a slider is audible instantly.
Keep **Write & Reboot** (`/save`) for persistence.

- **Live-safe** (read live in the loop): `clock_out_enable`, per-output
  `enable` / `cable` / `ppqn` / `phase_mbeats` (+ `swing` when P4-013 lands),
  `metronome_accent`.
- **Reboot-only** (never touched by `/live`): `wifi_ssid` / `wifi_pass`
  (reconnect), `metronome_enable` / `metronome_volume` / `metronome_voice`
  (ES8311 codec is initialized once at boot; no runtime setter).

### Mechanics

- Pure: split `ks_form` into `ks_form_apply` (patch ‚Äî set only the keys
  present in the body, leave the rest) and keep `ks_form_resolve` (full form ‚Äî
  absent checkbox => off) as `clear + apply`. `/live` uses the patch so a partial
  body doesn't disable everything.
- `/live` handler: recv ‚Üí `ks_form_apply(body, &g_cfg, &cand)` ‚Üí validate ‚Üí
  copy only the live-safe fields into `g_cfg` ‚Üí bump a config generation counter.
- Re-prime on change: a phase/ppqn/swing change shifts the grid, so the clock
  task must re-prime its tick/click grids or it dumps a one-time burst. The task
  watches the generation counter and resets the affected grids on change (they
  re-prime at the current beat ‚Äî silent realign). Cross-task safe: single-`int`
  field writes are atomic on the P4; `/live` writes fields, not a struct blob.
- Web UI: the NUDGE (and swing) controls POST to `/live` on input for real-time;
  Save still persists via `/save` + reboot.

## Deliverable

`ks_form_apply` + tests; `POST /live` glue; generation-counter re-prime in
`clock_out_task`; UI wiring. Host suite + `idf.py build` green. Verify by ear:
nudging one output audibly shifts its clock with no reboot; a full Save still
persists + reboots.
