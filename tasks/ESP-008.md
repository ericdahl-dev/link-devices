## Goal

Replace the onboard ES8311 mic with a real USB audio interface as the Link
Audio source, AND drive MIDI clock out through the same box -- Tier 4 of
`docs/plans/2026-07-08-p4-link-audio-feasibility.md`, scoped 2026-07-10 after
the PoC (P4-032) proved Tiers 1-3.

## Why one composite device

The P4 has ONE USB host port and no hub support (no transaction translator --
direct connect only, confirmed during P4-003). So "MIDI out + audio in"
requires a single composite device exposing both a USB-MIDI interface and a
USB-audio interface. That is also the product shape: one cable, KitchenSync
clocks the rig via the interface's MIDI OUT and ingests its preamps.

## Test device: XR18

Already owned, already the product's target mixer, and it is an 18-channel
USB audio interface WITH MIDI I/O. First candidate for the descriptor gate.
Consumer candidates for later: Behringer UMC204HD/404HD (cheap, same
ecosystem), MOTU M4, Steinberg UR22mkII -- all class-compliant with MIDI.

## The gate that decides everything

Espressif's `usb_host_uac` host component speaks UAC 1.0. Many interfaces are
UAC2-only (high-speed audio); some expose a UAC1 fallback configuration.
First step is a 10-minute descriptor dump on the P4:

- [ ] Plug the XR18's USB into the P4 Type-A, dump config descriptors
- [ ] Does a UAC 1.0 configuration exist? (UAC2-only = writing isochronous
      UAC2 host support -- a much bigger project; decide then)
- [ ] Does the MIDI interface enumerate alongside? (KitchenSync's existing
      USB-MIDI host path should find it in the composite)
- [ ] Check port power budget if the box is bus-powered (XR18 is not)

## What carries over unchanged from the PoC

beat_stamper (+servo), the LinkAudio sink path, session-rate sniff/reclock,
and the click-track calibration method. USB replaces `i2s_channel_read` as
the sample source; one click-track take re-derives the capture-latency
constant (USB adds its own constant pipeline delay).

## Acceptance Criteria

- [ ] XR18 (or another composite interface) enumerated by the P4: audio-in
      and MIDI-out interfaces both usable
- [ ] Live receives interface audio via Link Audio, click-track calibrated
- [ ] MIDI clock out through the same interface, phase-locked as today
