## Goal

When the device is following **Ableton Link**, emit an **analog clock pulse
train** (and a **reset** pulse, optionally a **run/start** gate) on GPIO →
level-shifted → 3.5 mm jacks, so a **Eurorack / analog modular** rig (or any
gear taking an analog clock: drum machines, sequencers, DIN-sync boxes) slaves
to the Link session. This is the analog sibling of **LNK-027** (Link → USB-MIDI
clock): same box, same tempo/phase pipeline, a different physical output. Three
sync fan-outs off one Link session: Link → OSC (mixer FX), Link → USB-MIDI clock
(LNK-027), Link → analog pulses (this).

## Why

The only way to clock a modular from Ableton Link today is a laptop + a
Link-to-CV interface (Expert Sleepers, a Pam's, etc.). A tiny ESP32-S3 that
already joins the Link session and already holds a bar-locked phase
(LNK-018/019/020, now accurate to ~−46 ms / ±14 ms after the LNK-026 fix) could
drive a modular clock input directly — no computer in the patch.

## Design sketch (not binding)

- **Shared tick source with LNK-027.** Both features want "emit an edge every
  1/N beat, first edge on phase-zero." Factor a single pure, host-tested
  scheduler (given `(bpm, phase, now_us, ppqn)` → ticks-due-now + next-due
  timestamp), consumed by the USB-MIDI path at 24 PPQN and by the analog path at
  a **configurable PPQN**. Gate on `tempo_source_phase_valid()` so we don't
  spray pulses before Link is measured; re-align on a timeline re-origin (reuse
  LNK-026's epoch-reset detection — a transport stop/start must not smear the
  clock).
- **Configurable clock rate.** Eurorack "clock" is most often **4 PPQN**
  (16th-notes); DIN-sync/Roland is **24 PPQN**; many modules want **1/2/8 PPQN**
  or an arbitrary division. Propose selectable `{1, 2, 4, 8, 24}` PPQN, default
  4. (Clock *division/multiplication* beyond integer PPQN is a stretch.)
- **Reset + run.** A **reset** trigger on the bar-1 / phase-0 crossing at the
  configured quantum (restarts downstream sequencers). A **run/start** gate
  (high while transport plays) is only possible with Link **StartStopSync** —
  same open item as LNK-027; clock+reset is the MVP, run is a follow-on.
- **Jitter discipline (critical — analog is audible).** Do NOT emit from
  `loop()` polling. Drive edges from a hardware peripheral — **RMT** is ideal
  for a precise pulse train, or MCPWM/LEDC/`esp_timer` — off a high-priority
  task. Modular clock consumers flam on sub-ms jitter.
- **Electrical (needs hardware, not just firmware).** ESP32-S3 GPIO is **3.3 V**;
  Eurorack triggers expect ~**5 V** (threshold ~1–2.5 V, tolerant but 3.3 V is
  marginal) and the patch can **back-feed ±12 V**. Level-shift 3.3→5 V (74HCT14/
  74HCT125 buffer on the USB 5 V rail, or an NPN/MOSFET stage) and protect each
  GPIO (series R + clamp diodes to the rail). Output on **3.5 mm mono jacks**
  (one clock, one reset, optional run). Use the board's **free header GPIOs
  IO1–IO11** (see the Waveshare pinmap memory) — avoid the LCD/touch/SD pins.
  Pulse width: short fixed trigger (~5–10 ms) or `min(10 ms, period/2)` so it
  stays valid at high PPQN × high tempo (24 PPQN @ 200 bpm → 12.5 ms period).
- **Config.** New NVS fields (`analog_sync_enable`, `analog_sync_ppqn`, maybe
  `analog_sync_pulse_ms`) + web-config and touch-UI toggles, only meaningful when
  `input_source == Link` (arguably also useful in USB-MIDI-in mode → analog out,
  decide during triage).

## Open questions

1. **Level-shifter approach** — buffer IC vs discrete transistor; how much GPIO
   protection is enough for a hostile modular patch. This is the real design
   risk (the firmware is mostly LNK-027 reused).
2. **Default PPQN + which divisions** to expose; is fixed `{1,2,4,8,24}` enough
   or do users need arbitrary clock division/multiplication?
3. **Reset semantics** — bar reset (phase-0 at quantum) vs a one-shot on
   session join; how it should behave across a tempo change / re-origin.
4. **Run/start gate** — clock+reset only, or wire Link StartStopSync (shared
   decision with LNK-027)?
5. **DIN-sync (Sync24)** — worth a 5-pin DIN variant (24 PPQN + run/stop on one
   connector) for Roland-style gear, or keep it to bare 3.5 mm clock/reset?
6. **Which board** — the Waveshare touch board has free GPIOs but no jacks; this
   implies a small add-on/protoboard or a board revision. Scope the hardware
   deliverable, not just firmware.

- [ ] New NVS config (`analog_sync_*`) exposed in the web config UI and the touch
      settings screen; only actionable when tempo source = Link.
- [ ] Tick-schedule math lives in a pure host-tested module wired into

- [ ] Verified on hardware: a real modular (or a scope/logic analyzer) shows the
      clock locked to the Link bar with no visible jitter/flam, and reset firing
      on the downbeat, across a transport stop/start and a tempo change.
- [ ] No regression to Link → OSC, USB-MIDI-in, or (if landed) LNK-027 MIDI-out.

## Notes

- Strongly coupled to **LNK-027** — land the shared scheduler once and let both
  the USB-MIDI and analog outputs consume it, rather than two tick engines.
- Rides on the phase pipeline validated in **LNK-020** and the transport-re-origin
  handling from **LNK-026** (an analog clock that smears on every stop/start is
  worse than none).
- Filed from a feature ask (2026-07-03); hardware design (level-shift + jacks) is
  the gating unknown, hence `priority: low` / `needs-triage` until the board
  question (add-on vs revision) is decided.
