## Goal

Upgrade the P4-005 clock from **correct-tempo** to **phase-locked**: the emitted
clock/downbeat aligns to the Link session's actual beat/bar position, so a MIDI
Start (P4-008) lands on beat 1 and multiple synced devices agree on the downbeat —
not just the rate.

## Why

P4-005's `beat_clock` is anchored to the *local* clock (correct rate, arbitrary
phase) because the Link timeline's `time_origin_us` is in the peer's ghost clock,
not ours. Aligning to the session beat needs the local↔ghost time mapping.

## Reuse

The S3 already has this pipeline, pure and host-tested: `link_measurement.c` +
`link_measurement_session.c` (ping/pong RTT → GhostXForm) and
`link_phase.c:link_phase_beats_now(timeline, ghost_now_us)`. Port the *glue*
(`link_measurement_io.cpp` unicast WiFiUDP → ESP-IDF sockets to the peer's mep4
measurement endpoint); the math is reused unchanged.

## Scope

- ESP-IDF unicast ping/pong glue to the Link peer's measurement endpoint
  (reuse the session orchestrator + measurement math).
- Feed `link_phase_beats_now()` with `host→ghost`-mapped time instead of the
  free-running `beat_clock`; keep re-origin/epoch-reset handling (LNK-026).

## Deliverable

The P4's clock downbeat tracks the Link session beat/bar (verify against Ableton
+ a second synced device). Reuses the S3's measurement/phase modules unchanged.

## Progress (feat/p4-009-phase-lock)

Implemented. Files:

- **NEW** `KitchenSync/main/link_measure_io.{c,h}` — ESP-IDF/lwip port of the S3's
  `link_measurement_io.cpp`. Unicast UDP socket (ephemeral local port,
  non-blocking) driving the pure measurement session. Opens lazily on first poll;
  `link_measure_io_poll()` runs in `clock_out_task`'s 1 ms loop.
- **CHANGED** `KitchenSync/main/ks_main.c` — `clock_out_task` now pumps
  `link_measure_io_poll()` and, when `link_measurement_current_xform().valid`,
  feeds `clock_ticker` from `link_phase_beats_now(timeline, host->ghost(now))`
  (true session beats). Falls back to the free-running `beat_clock` until the
  first GhostXForm commits; re-primes the ticker on the free<->locked switch.
  Epoch-reset (LNK-026) is handled inside the reused session orchestrator
  (`LS_RESET_XFORM`).
- **CHANGED** `KitchenSync/main/CMakeLists.txt` — adds `link_measure_io.c` +
  reused-unchanged `link_measurement.c`, `link_measurement_session.c`,
  `link_phase.c` via relative-path SRCS.

Reused UNCHANGED (no re-implementation): `link_measurement.c`,
`link_measurement_session.c`, `link_phase.c`, `clock_ticker.c`, `beat_clock.c`,
`link_protocol.c`. No NEW pure logic was needed (host->ghost map, phase-beats,
epoch-reset all already exist + are host-tested), so no new Unity suite was added
