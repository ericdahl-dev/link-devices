## Goal

Write the final `X32MidiClock.ino` main sketch that wires USB MIDI clock
receive → BPM calculation → OSC send into a coherent FreeRTOS task loop.
Mirrors the structure of `X32Link.ino` but replaces the Link listener with
the MIDI clock module.

## Acceptance Criteria

- [ ] `setup()` sequence: serial, factory-reset check, config load, WiFi connect
      (or AP fallback), `midi_clock_init()`, `osc_sender_begin()`, `bpm_tracker_init()`,
      `web_config_begin()`, spawn `bpm_task`
- [ ] `bpm_task` polls `midi_bpm_update()` every `MCK_POLL_MS` (≤ 10 ms)
- [ ] When BPM > 0 and `bpm_tracker_update()` returns true, calls `osc_send_bpm(bpm)`
- [ ] When BPM returns to 0 (clock stopped / timeout), does NOT send OSC — X32
      keeps its last tempo until clock resumes
- [ ] `loop()` handles WiFi reconnect + `web_config_handle()` + heartbeat log
- [ ] Heartbeat Serial log: `[X32MidiClock] bpm:120.00 pulses:2400 heap:210000`
- [ ] No WDT resets under normal operation; no crash on USB hot-plug/unplug

## Notes

**Task structure** (mirrors X32Link):
```cpp
static void bpm_task(void*) {
    for (;;) {
        float bpm = midi_bpm_update();
        if (bpm > 0.0f && bpm_tracker_update(bpm)) {
            osc_send_bpm(bpm);
            g_current_bpm = bpm;
        }
        vTaskDelay(pdMS_TO_TICKS(10));   // 10 ms poll — fast enough, cheap
    }
}
```

**USB hot-plug**: TinyUSB handles host connect/disconnect gracefully. The ring
buffer just stays empty when disconnected; `midi_bpm_update()` returns 0.0 after
the timeout, and OSC stops. On reconnect, Ableton re-sends clock and the
firmware picks it back up within one beat.

**Clock-stop behaviour (design decision)**: When the musician stops the DAW,
we intentionally do NOT reset the X32 tempo to zero or a default. The desk
keeps the last known tempo — the band can keep playing in time if the DAW
stops unexpectedly. Tempo only changes when a new valid BPM arrives.

**MCK_POLL_MS**: Use 10 ms (vs Link's 50 ms). At 240 BPM, pulses arrive every
10.4 ms — polling at 10 ms ensures we never miss processing a fresh pulse batch
within the same FreeRTOS tick as the callback.
