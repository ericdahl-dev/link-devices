## Goal

Create the `esp32/X32SafeMutes/` Arduino sketch — an ESP32-S3 firmware that
guards selected channel mutes: if a *locked* channel's mute state changes
unexpectedly, it is immediately restored (FDR logic lands in MUT-002). Works on
both **X32 (10023)** and **XR18 (10024)** — mutes exist on both.

This task is scaffold + handshake + receive only.

## config.h contents

```c
#pragma once
#define MUT_CHAN_COUNT       32     // X32 (16 for XR18)
#define MUT_RESTORE_DELAY_MS 50     // debounce before restoring a locked mute
#define MUT_XREMOTE_INTERVAL 9000   // ms between /xremote keepalives
// Locked-channel bitmask default; overridden by web UI / NVS:
#define MUT_DEFAULT_LOCK_MASK 0x00000000UL
```

## Notes

- Mute semantics are **inverted**: `/ch/NN/mix/on = 1` is UNMUTED, `0` is MUTED
  (see [`docs/xr18-xair-osc-cheatsheet.md`](../../docs/xr18-xair-osc-cheatsheet.md)).
- No restore logic in this task — just prove the change-detection feed works.
