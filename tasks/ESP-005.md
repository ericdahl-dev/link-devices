## Goal

Make X32 state save/load (X32Init, X32Shutdown) work on ESP32 via LittleFS VFS rather than the local filesystem.

## Acceptance Criteria

- [ ] LittleFS mounted in X32_emulator.ino before x32_init() is called
- [ ] All `fopen(".X32res.rc", ...)` in X32.c copy replaced with `fopen(X32_STATE_FILE, ...)`
- [ ] X32_STATE_FILE = "/littlefs/.X32res.rc" on ARDUINO (defined in x32_port.h)
- [ ] X32_STATE_FILE = ".X32res.rc" on Linux (existing behavior unchanged)
- [ ] Native tests still pass
- [ ] Sketch compiles with no path-related errors
- [ ] On-device: /shutdown OSC command saves state; power-cycle restores it

## Notes

**File to edit**: esp32/X32_emulator/X32.c (copy)

**fopen occurrences** (original X32.c):
- Line 5400: `fopen(".X32res.rc", "w")` — X32Shutdown save
- Line 5543: `fopen(".X32res.rc", "r")` — X32Init load

**Change**: s/`".X32res.rc"`/`X32_STATE_FILE`/g — two replacements only.

**LittleFS VFS**: ESP32 Arduino 3.x mounts LittleFS at `/littlefs/` via VFS.
POSIX fopen/fclose/fprintf/fscanf all work through this mount point.
No rewrite of file I/O logic needed — only the path string changes.

**Mount in X32_emulator.ino**:
```cpp
#include <LittleFS.h>
// in setup():
if (!LittleFS.begin(true)) {  // true = format on first use
    Serial.println("LittleFS mount failed");
    while(1);
}
```

**Partition scheme**: default_8MB gives ~1.5MB SPIFFS/LittleFS — plenty for X32 state file.
